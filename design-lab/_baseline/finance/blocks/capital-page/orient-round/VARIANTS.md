# orient-round — ориентация на странице капитала

**Продуктовые решения (2026-05):**

| Решение | Значение |
|---------|----------|
| Приоритет | **Разобраться** в потоке; «сделать» — только после раскрытия раздела |
| Фокус раунда | **Доходы / Расходы** + сводка сверху |
| Бюджет | **Не в плане** (отдельная вкладка убрана) |
| Цель сценария | **Нет** ссылки с этой страницы |

## Варианты

| ID | Идея | Плюсы | Минусы |
|----|------|-------|--------|
| **O1 ★** | Сводка + доходы/расходы open; **подсказка** между потоком и разделами действий | Стили строк как flows-round; только вставка текста | Длинный первый экран |
| **O2** | Один аккордеон «Поток периода» (доходы+расходы+итог) | Меньше заголовков, одна история | Тяжелее сканировать только доходы или только расходы |
| **O3** | Два аккордеона + **мост** «после расходов −11 500» | Связка двух блоков, просрочка в сводке | Дублирование цифры потока (сводка + мост) |
| **O4** | Сводка + только **доходы** open, расходы tease | Компактный вход | Расходы спрятаны — нужен второй тап |

## Сводка (`.cap-health`)

Три ячейки (не victory):

- **Поток** — `net_monthly_cashflow` (как chip «Доходы» на дашборде, но подпись «Поток»)
- **Подушка** — % от нормы **×3 текущих обязательств** (полоска как на дашборде)
- **Просрочка** — сумма или «нет»

## Prod (после утверждения)

- `CapitalPeriodFlowsBlock` + новый `CapitalHealthStrip` в `FinancePremium`
- Дефолт: O1 — оба потока open при первом заходе; с дашборда — open только нужный (`openFlowsSection`)
- Стили: `mqx-cap-health*`, переиспользовать `mqx-cushion-fill*`
