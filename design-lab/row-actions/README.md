# Row actions — кнопки + / − в списках

Этап 1 MQX: варианты до React в prod.

## Утверждение

**2026-05-19 — вариант B** (компактный glass −, hover на desktop, hit-area 44px).

Продуктовые решения:

- Подтверждение **для всех** разрушающих действий (пока без исключений).
- Список **позиций** — компактные строки с **−**, не карточка «Удалить».
- Plan (`BaseParamsScreen`, `PlanExpenseEditor`) — **не трогаем** в этой волне.

## Варианты

| ID | Идея | Статус |
|----|------|--------|
| A | Широкая текстовая «Удалить» (legacy `mqx-capital-delete-btn`) | отклонён |
| B | Компактный −, glass, hover danger | **утверждён → prod** |
| C | − в круглом outline, сильнее контраст | запасной |
| D | + violet filled / − ghost (как сейчас mix в Finance) | референс |

## Запуск

```bash
cd design-lab/row-actions
npx serve .
```

После внедрения: `npm run dev` → `#/dev/mqx` → «Паттерны действий».
