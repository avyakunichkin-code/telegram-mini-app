# Onboarding — Mission Brief + Монетка

**Статус:** раунд 1 — **выбор варианта** (prod не трогаем).

**Решения продукта (2026-05-19):**

- Показ **после загрузки overview** на `GameScreen` (оверлей поверх игры).
- **Волна 1:** только Mission Brief; coach marks — отдельная волна после утверждения.
- Вторая новая игра — бриф **не** показывать; **«Повторить обучение»** в меню.
- Тон **игровой**; наставник — **Монетка**.

Запуск:

```bash
cd design-lab/onboarding-brief
npx serve .
```

## Варианты (раунд 1)

Одинаковые данные: **шаг 2 из 3** («Зарплата и подушка»), кнопки **Пропустить** / **Далее** / точки 1–2–3.

| ID | Идея | Монетка | Когда брать |
|----|------|---------|-------------|
| **A** | Только текст, flat `mqx-card`, violet kicker | нет | Минимум UI, быстрый Pre-Alpha |
| **B** | Bubble сверху, Монетка по центру над карточкой | крупно | Классический «наставник говорит» |
| **C** | Монетка в углу chip + компактная карточка | малый avatar | Больше места тексту, меньше «мультик» |
| **D** | «Дневник квеста» — mist-фон, kicker «Глава 1/3» | мини в шапке журнала | Сильнее ощущение игры/кампании |
| **E** | Верхняя «сцена» (градиент violet) + карточка снизу | в сцене | Эмоциональный акцент, чуть тяжелее |
| **F** | Горизонтальная лента 3 мини-панелей (прогресс главы) | на каждой панели | Обзор всего брифа сразу; риск мелкого текста |

**Рекомендация для обсуждения:** **B** или **C** для TMA 320px; **D** если важнее «кампания»; **A** как fallback без персонажа.

## Как утверждать

1. Пройти все варианты в светлой и тёмной теме.
2. В чате: «Утверждаем **X**» (или гибрид: «**C** + сцена из **E**»).
3. Обновить [`CHARACTER_MONETKA.md`](../../docs/reference/CHARACTER_MONETKA.md) § Утверждённый вариант.
4. Только после этого — MQX + API (см. [`PLAN_onboarding-tma.md`](../../docs/plans/PLAN_onboarding-tma.md)).
