# goal-chain-round — цепочка целей на дашборде

## Смена парадигмы (продукт)

| Было | Стало |
|------|--------|
| Несколько целей параллельно, победа при M из N | **Цепочка**: одна активная цель, после выполнения — следующая |
| Набор целей общий / M из N | **Свой набор цепочки у каждого `game_starter_template`** |
| Блок «Период N» + шкалы прогресса | Раздел **«Цель»**, **без шкал** внутри |
| Список целей с `MqxProgress` | **«Как достичь»** — текст, `<strong>` для ключевых цифр и действий |

## Порядок на дашборде (S5)

1. Финансы периода  
2. **Цель** (аккордеон)  
3. Действия периода  

*(Уровень/XP и «Достижения» — вне этого раунда; старый `MqxLevelDash` заменяется/переименовывается.)*

## Варианты UI

| ID | Свёрнутый заголовок | Развёрнутое тело | Статус |
|----|---------------------|------------------|--------|
| **G1 ★** | Kicker «Цель», название текущей цели, «Шаг K из N» | **Монетка** + «Давай, помогу разобраться» + цепочка; без дубля заголовка и badge | **Утверждено (lab v2)** |
| G2 | То же | Только «Как достичь», **без** списка цепочки | Минималистичный fallback |

## Свёрнутое состояние G1

- **Нет** `MqxProgress` и «Цели: M из N».
- Есть chevron, градиентный bleed как у level-dash (emerald → violet).

## Развёрнутое состояние G1 (v2)

- **Нет** повторного заголовка цели и **нет** badge статуса.
- **Монетка** + пузырь, заголовок **«Давай, помогу разобраться»**, текст **10px**, `<strong>` — голубой акцент (sky).
- **Цепочка сценария** внизу: ✓ / текущий шаг / locked, без progress bar.
- Цвет раздела: **голубой** bleed (как level/XP), не emerald.

## Prod (после утверждения)

- Backend: `goal_chain` / расширение `victory_config`, API `overview.victory` → `current_goal`, `chain[]`, `guidance`
- Frontend: `MqxGoalDash` вместо `MqxLevelDash`, перестановка в `DashboardPremium`
- Контент подсказок: по `type` цели или явные строки в конфиге шаблона
