# Period actions round — варианты

**Раунд:** `design-lab/dashboard/period-actions-round/`  
**Контекст:** S5 unified, блок L3 №3 (после «Цель»).

## Зафиксировано (вход)

| Решение | Деталь |
|---------|--------|
| Подзаголовок | Убрать «Управляй денежным потоком» |
| Монетка | Другая поза (`think` / `wink`), **справа** от пузыря, спокойный sky-фон |
| Зарплата получена | **Disabled chip**, текст «Зарплата» не меняется |
| Паттерн | Chip как у «Финансы периода», не `mqx-action` 2×2 |

## Варианты в витрине

| ID | Суть | Плюсы | Минусы / риски |
|----|------|-------|----------------|
| **R0** | Текущие кнопки + card-sub | — | Несогласованность с S5 ★ |
| **P1 ★** | 4 chip: Зарплата · На подушку (с карты) · На счёт (из подушки) · Капитал → | Ясные направления; 4 ячейки; без новой механики | «Капитал» — переход, не действие периода |
| **P2** | Hub «Подушка» + sheet; Депозит · Облигация | Одна плитка вместо «В подушку»/«Снять»; инвестиции явно | +1 тап на подушку; нужен UX sheet в prod |
| **P3** | Глаголы «Перевести» + sub | Короткие подписи | Два одинаковых «Перевести» — путаница |
| **P4** | 3col + 2 chip вложений | Компактно по вертикали | Узко на 320px; двухрядность |

## Механика подушки (обсуждение)

| Подход | Когда уместен |
|--------|----------------|
| **Две chip (P1)** | Как сейчас в API: `contribute` / `withdraw` — минимум сюрпризов |
| **Hub + sheet (P2, P4)** | Если важнее сэкономить место и объяснить в Монетке |
| **Одна chip + существующая панель суммы (prod)** | Hub открывает тот же inline-slider, что сейчас под кнопками |

## Монетка

- **P1, P3:** `assets/monetka-think.png` — спокойная подсказка.
- **P2, P4:** `assets/monetka-wink.png` — лёгкий тон при hub/вложениях.
- Пузырь **слева**, хвост **вправо** к маскоту (не как finance-monetka).

## Рекомендация для ★ (черновик)

1. Взять **P1** как визуальный и текстовый базис (chip + подписи направления).
2. Если нужны **депозит/облигация на дашборде** — гибрид **P1 + P2**: hub подушки **или** две chip подушки (как P1), вторая строка только «Депозит» / «Облигация» (как P4).
3. Prod: перенос в `MqxPeriodActions` → `MqxPeriodActionChip` + `MqxPeriodHint` после утверждения.
