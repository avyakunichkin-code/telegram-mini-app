# Z-NEEDS v6 — варианты

**Раунд:** `dashboard-needs-v6-round`  
**База:** v5 ★ prod ([`../dashboard-needs-v5-round/APPROVED.md`](../dashboard-needs-v5-round/APPROVED.md))

| ID | Название | Идея | Плюсы | Риски |
|----|----------|------|-------|-------|
| **v5** | Эталон | Одна min-ось в collapsed | Привычно, уже в prod | Не видно остальные 3 оси без expand |
| **v6-A** | Glass + quad | Стеклянная карточка, 2×2 мини-шкалы | Все 4 потребности с первого взгляда; ближе к S5 glass | Выше блок; плотнее в collapsed |
| **v6-B** | Ring + % | Кольцо статуса, kicker, % у min-оси | Эмоциональная связь с персонажем; точнее цифра | Кольцо может спорить с портретом; footer сложнее |
| **v6-C** | Segment strip | 4 сегмента в одной полосе | Самый низкий по высоте | Сегменты не точные % (визуальный вес); меньше подписей осей |

## Рекомендация для обсуждения

- Если цель — **больше информации без expand** → **v6-A**.
- Если цель — **характер + срочность** → **v6-B**.
- Если цель — **компактность дашборда** → **v6-C** (или гибрид: v6-C collapsed + v5 expanded).

## Инварианты (не ломаем)

- Позиция Z-NEEDS после hero (UX-01).
- Bleed risk banner при `streak > 0`.
- Expand: 4 шкалы + «Порадовать себя» + «?».
- Текстовые дубли зон (не только цвет).
- `role="progressbar"` на шкалах.
