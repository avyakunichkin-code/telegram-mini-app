# Главная (dashboard) — финальный раунд компоновки

**Цель:** закрыть IA Главной до prod: прогресс (уровень + победа + достижения) vs операции vs капитал (в т.ч. **компактные формы** на главной).

**Spec-контекст:** [`SPEC_achievements.md`](../../docs/specs/features/SPEC_achievements.md) §11, [`shell/APPROVED-LAYOUT.md`](../shell/APPROVED-LAYOUT.md) (D′ — референс, не обязательно финал).

## Запуск

```bash
cd design-lab/dashboard-home-v2
npx serve .
```

Светлая / тёмная тема — в шапке. Одинаковые тестовые данные во всех вариантах (кроме блока **F**).

## Матрица выбора (быстро)

| Критерий | Сильные кандидаты |
|----------|-------------------|
| Зарплата за 3 сек | V3, V4, V9, V13 |
| Формы капитала без вкладки Finance | V3, V7, V12, V14 |
| Прогресс отдельно от операций | V2, V11, V14 |
| Минимум скролла на 390px | V2, V9, V12 |
| Не accordion | V2–V5, V8–V14 |
| Гибрид «лучшее из двух» | V14 (V2+V3), V11 (rail+forms) |

## Критерии утверждения

1. **Первый экран** — действие + сигнал прогресса без лишнего скролла.
2. **Зарплата** заметна сразу.
3. **Капитал** — шаг (вклад / платёж) на главной *или* 1 тап до Finance.
4. **Прогресс** не смешивается с «score» финграмотности.
5. Не более **2** primary CTA одновременно.

## Утверждение

После выбора — `APPROVED.md` + обновление spec + MQX → `DashboardPremium`.
