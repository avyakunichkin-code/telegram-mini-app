# Раунд 1 — компоновка карточки события

**Цель:** выбрать **каркас** (слоты и порядок), не финальные цвета доменов.  
**Данные:** одинаковые для всех вариантов — «ДТП» (страховой) и «Кофе каждый день» (consumption).

## Слоты карточки (что решaем)

| Слот | Варианты |
|------|----------|
| **Домен** | Полоска сверху / pill в шапке / кольцо у Монетки / только иконка |
| **Контекст** | ~~Kicker периода~~ — убран в L3 ★; период виден в shell игры |
| **Заголовок** | H3, одна строка |
| **Нарратив** | Пузырь + Монетка (M2) / plain text / clamp 2 строки |
| **Монетка** | Всегда / только `meta`+`social` / никогда (домен-иконка) |
| **Выборы** | Flat-список (канон prod), impacts в строке |

## Варианты

| ID | Название | Идея | Монетка | Домен в R1 |
|----|----------|------|---------|------------|
| **L1** | M2 ★ prod | Пузырь + Монетка слева | Да | Badge страховки только |
| **L2** | M1 compact | Монетка 58px + текст справа, выборы снизу | Да | Chip в meta |
| **L3** | Domain band ★ | 4px полоса + pill в ряд, title + bubble | Да | **Полоса + pill** |
| **L4** | M4 rail | Кольцо слева, домен в кольце | Да | **Кольцо** |
| **L5** | Title-first | Pill у title, описание без пузыря | Нет | **Pill + tint фона** |

## Запуск

```powershell
cd design-lab/events/layout-round

```

Self-contained: только `./` в HTML. Общий sync: `design-lab/events/sync-all-rounds.ps1`. Правило: `tvoy-hod-design-lab.mdc`.

## После ★ компоновки

## Критерии выбора

1. Читаемость на 320px, 3 выбора без лишнего скролла.  
2. Домен узнаётся за 1 с (полоса/pill/кольцо).  
3. Монетка не на каждом событии — если L5, нужен fallback-икон домена.  
4. Страховой случай не ломает layout (модификатор, не отдельная вёрстка).
