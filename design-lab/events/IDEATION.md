# Карточка события + Монетка — ideation (2026-05-19)

## How Might We

**Как сделать карточку события компактнее и живее, чтобы игрок быстрее считывал ситуацию и выбор, не перегружая экран — с Монеткой (PNG, прозрачный фон) как визуальным якорем, а не комментатором каждой строки?**

## Для кого и успех

| | |
|--|--|
| **Кто** | Игрок в TMA, 1–3 события за период, мобильный экран |
| **Успех** | Меньше вертикального скролла; заголовок + суть за &lt;3 с; выборы остаются tappable; страховой случай визуально отличим |
| **Ограничения** | MQX, Quest Violet / emerald; одна PNG Монетки (без поз v1); не ломать карусель и свайп |

## Кластеры направлений (после diverge)

| Кластер | Варианты | Суть |
|---------|----------|------|
| **Компакт + якорь** | M1, M5 | Монетка маленькая слева/в шапке; текст сжат (line-clamp); выборы flat |
| **Нарратив** | M2 | Описание = «реплика» в пузыре; заголовок события снаружи |
| **Визуальный ивент** | M3, M4, M6 | Градиент, кольцо, halo; сильнее «игровая карточка» |
| **Домен страховки** | M6 (+ M3 insurance) | Emerald, бейдж, пульс — без смены логики выборов |

## Варианты в design-lab

| ID | Название | Монетка | Компактность | Визуал |
|----|----------|---------|--------------|--------|
| **M1** | Компактный ряд | 58px, тень | ★★★ | chip периода, 2 строки desc |
| **M2** | Реплика в пузыре | 50px, хвост bubble | ★★ | coach-стиль, violet/emerald bubble |
| **M3** | Шапка-полоса | 72px на границе hero | ★★ | gradient hero, icon pills |
| **M4** | Колонка-визуал | в кольце 68px | ★★★ | rail слева, текст справа |
| **M5** | Ультра-компакт | 44px | ★★★★ | рамка-карточка, мелкие выборы |
| **M6 ★** | Страховой бриф | 64px + halo | ★★ | emerald panel, pulse, badge |

**Утверждено (2026-05-19):** **M2** в prod. Все выборы flat, без emerald primary на одном пункте.

**Было рекомендовано:** гибрид M1 + M6 — отклонено в пользу M2.

## Стресс-тест

| Критерий | M2 (пузырь) | M1/M5 (компакт) |
|----------|-------------|-----------------|
| User value | Эмоция, «напарница» | Скорость, меньше высоты |
| Feasibility | Средне (копирайт = desc API) | Высокая |
| Differentiation | Похоже на онбординг | Уникально для events |

**Риск:** M2 на 50+ событиях ощущается однообразно («все говорит Монетка») — против канона «не комментирует каждое событие». **Митигация:** M2 только как визуальная оболочка `description`, без отдельного tone-of-voice.

## Assumptions

- [ ] Игроку не мешает Монетка на каждой карточке (A/B в playtest)
- [ ] line-clamp 2 для desc достаточен (длинные тексты редки)
- [ ] 44–64px PNG читается на 320px ширине

## MVP scope (после утверждения)

**In:** выбранный вариант(ы) в `EventCard.jsx`, стили `mqx-events-card--m*`, `MonetkaAvatar` reuse, страховой модификатор.

**Out:** отдельные PNG-позы; анимация Монетки; разный текст от Монетки vs event description.
