## 1. Концепция игры и базовые механики

### 1.1. Жанр и формат

**ТВОЙ ХОД** — Telegram Mini App: **игра по финансовой грамотности**, а не сухой курс. Пошаговый формат: **один ход = один игровой период** («месяц»); период закрывается кнопкой **«Закрыть месяц»**, без real-time таймера ([`SPEC_PRODUCT` §3.1](docs/foundation/SPEC_PRODUCT.md)). Игрок проживает решения в **безопасной песочнице** с правилами, близкими к реальности.

**Педагогический тезис:** часть ошибок допустима и учит (например, не забрать зарплату до конца периода — выплата за период не повторяется).

### 1.2. Основные финансовые инструменты

| Инструмент | Смысл в игре | MVP |
|------------|--------------|-----|
| **Зарплата** | Основной приток; только по явному действию в периоде | ✅ |
| **Кредиты / обязательства** | Платежи, просрочка, тело на счёт при оформлении | ✅ |
| **Недвижимость, транспорт** | Активы: стоимость, обслуживание, доход (аренда) | ✅ из шаблонов |
| **Вклад (deposit)** | Фиксированная ставка, капитализация в `principal` | ✅ |
| **Облигации (bond)** | Купон на cash; досрочное закрытие | ✅ |
| **Акции, ETF, биржа** | Высокая волатильность, портфели | ❌ **вне scope** |
| **Налоги, ИИС, пенсия** | Макро и долгий горизонт | ⬜ backlog |

**Детали из анкеты (упрощённо):** вклад — предсказуемость; облигации — выше доход, риск при досрочной продаже; недвижимость — жить vs сдавать; транспорт — чаще расход, опционально карьера.

### 1.3. Механики стaбильности и защиты

- **Подушка безопасности** — ликвидный резерв (ориентир 3–6 мес. расходов); отдельный баланс `safety_fund_balance`.
- **Страховки** — регулярная премия vs защита от редких крупных убытков (здоровье, имущество, авто).

### 1.4. Базовый игровой цикл

**Фазы периода (как в анкете, адаптировано под код):**

1. **Доходы (в т.ч. по действию игрока):** зарплата по кнопке; в конце периода — проценты вклада, купоны облигаций, доход активов.
2. **Автоматические списания:** обслуживание активов, платежи по долгам (+ просрочка), премии страховок, базовые «жизненные» расходы (`base_monthly_lifestyle_expense` + дельта).
3. **События:** **две** карточки с выбором за период; эффекты на cash, подушку, lifestyle (`xp_delta` в сидах **игнорируется**).
4. **Действия игрока (несколько):** подушка, покупки из каталогов, инвестиции, страховки — пока открыт период.
5. **Конец периода:** `process_period_end` → новый `period_index`, снимок аналитики, генерация событий следующего периода.

### 1.5. Активы и пассивы

Учить отличать **актив, работающий на капитал**, от **потребительской траты**. Blueprint может выдать ипотеку с первого дня; **новые** разделы капитала (инвестиции, страховки…) — через **`mechanics_unlock`** после шагов цели победы ([ADR-004](docs/decisions/ADR-004-mechanics-unlock-victory-chain.md)), не через character level.

### 1.6. Кредитование

Параметры: сумма, срок, ставка, тип платежа (аннуитет / дифференцированный — целевое обогащение), досрочное погашение, переплата за срок. Риски: просрочка, рост `overdue_amount`, ухудшение условий. В UI — **суммы**, не формулы.

### 1.7. Подушка и страховки (обучающий смысл)

Не приумножение, а **снижение волатильности**: без резерва шоки толкают в дорогой долг или продажу активов в невыгодный момент.

### 1.8. Цели и победа

**Prod:** [`victory_engine`](backend/app/victory/engine.py) + `victory_config_json` шаблона ([ADR-002](docs/decisions/ADR-002-victory-engine-and-template-config.md), [SPEC_victory-v2](docs/specs/features/SPEC_victory-v2.md)):

- **`progression_mode: chain`** (tutorial на всех Game-шаблонах): все шаги цепочки выполнены + `period_index >= min_period_index_for_victory` (обычно **7**).
- **`progression_mode: parallel`** (откат): **M из N** среди enabled-целей + ворота периода.
- UI: `MqxGoalDash` + `overview.victory`.

**Legacy (только unit-тесты):** `evaluate_mvp_victory` — AND «подушка 3× + нет просрочки + cashflow ≥ 0».

Детали пулов целей и cashflow — [`tvoy-hod-evolution-after-mvp.md`](docs/vision/ideas/tvoy-hod-evolution-after-mvp.md) §II.

Дополнительные **концептуальные** критерии из анкеты: финансовая независимость (пассив > расходов), чистый капитал до порога, прохождение без «долговой ямы», индекс финансового здоровья.

### 1.9. Образовательные слои

- Индикатор инфляции / рост «жизни» (lifestyle + события).
- Экран аналитики: доля долга в доходе, подушка в месяцах, cashflow, timeseries — [`SPEC_ANALYTICS.md`](docs/specs/SPEC_ANALYTICS.md).
- Короткие предупреждения при опасных паттернах.
- Макро-события (ставка, кризис) — со 2-й половины прогрессии (backlog).

### 1.10. Режимы сохранения: Game и Plan

- **`save_kind: game`** — старт из **`game_starter_templates`**, blueprint, без ручного ввода базовых параметров.
- **`save_kind: plan`** — ручное планирование, статьи расходов, префилл из снимка (MVP 2.0); в UI пока «Скоро».
- Legacy **`light` / `hardcore`** сняты — [ADR-001](docs/decisions/ADR-001-save-kind-remove-light-hardcore.md).

## 2. Геймдизайнерские вопросы перед разработкой

Вопросы из анкеты — **ориентиры дизайна**; ответы ниже с учётом текущих решений.

### 2.1. Эмоциональная цель

Доминирующий тон: **спокойствие и уверенность** при подушке и дисциплине; напряжение при долге — через UI (цвет, тосты, overview), не через запрет информации.

### 2.2. Симулятор vs игра

Ключ — **компромиссы внутри периода**: зарплата до закрытия месяца, **два** события vs cash, вклад vs подушка. *Не* «один выбор за месяц» (отказ от анкеты).

### 2.3. Визуализация абстракций

Метафоры (цепь долга, щит подушки, дерево вклада) **дополняют** цифры. Без читаемых сумм в точках решения игрок теряет контроль — учтено в ТВОЙ ХОД.

### 2.4. Бесконечная цель и прогрессия

Победа из шаблона + рост сложности событий по **периоду** (`event_tier`) + **новая партия** с другим шаблоном. ~~`character_level`~~ — снят.

### 2.5. События без «гнева рандома»

Предвестники (целевое), tier-окно от периода, веса, **cooldown**, обязательные вехи на первом проходе (backlog).

### 2.6. Второй шанс

Поражение MVP; новая игра; в перспективе — «банкротство» с ограничениями или разовая помощь с последствиями (из анкеты).

### 2.7. Против стратегии «зажать всё»

Параметр комфорта / события-искушения / lifestyle — не через искусственный лимит одного действия.

### 2.8. Мета-игра

**Реализовано:** шаблоны старта, список профилей, реиграбельность.

**Из анкеты (vision):** сценарии «В наследство», «Кредитное бремя», «Фрилансер», испытания; коллекционные «Уроки жизни»; «Свободный режим» после кампании.

## 3. Баланс лёгкости и образовательного аспекта

Принципы из анкеты с адаптацией под ТВОЙ ХОД.

### 3.1. Числа и визуал вместе

*Анкета:* «каждое число — визуальный объект». **Сейчас:** иконка + подпись + **сумма/процент**; полоски для трендов; карточки инструментов с 2–3 свойствами **и** цифрами из API.

### 3.2. Несколько действий за период

Группировка UI: шапка (время, события), дашборд, финансы (вкладки), периодные CTA. Минимум для «логичного» закрытия периода в MVP — **зарплата**; остальное — стратегия.

### 3.3. Инструменты как «карты»

🛡️ Надёжность | 💰 Доход | 🔒 Ликвидность | ⚡ Риск + одна фраза «живым языком» + **ставка/платёж**. Симуляция последствия до подтверждения (целевое усиление ползунков).

### 3.4. Обучение через ситуации

Поэтапное открытие механик через события (друг занял, подарок — вклад или cash, зуб без страховки). Контекстная подсказка «?» — backlog.

### 3.5. Мгновенная обратная связь

Тосты, overview после выбора, level-up; «настроение мира» от долга/подушки — целевое усиление визуала.

### 3.6. Ползунки и переключатели

**Утверждено:** ползунок + число + превью (платёж, переплата, остаток cash). Формулы — на сервере.

### 3.7. Дневник персонажа

Необязательные записи постфактум («если бы застраховался…») — backlog; не блокируют основной поток.

## 4. Структура игры: шаблоны, главы, сессии

### 4.1. Два слоя структуры

1. **Реализовано сейчас:** каталог **`game_starter_templates`** — разная сложность, blueprint, `base_monthly_lifestyle_expense`, задел `victory_config`.
2. **Vision из анкеты:** «Главы жизни» — сценарии на 12–36 месяцев с фокусом на 1–2 темы финграмотности за главу.

### 4.2. Пример главы «Мечта о буфере» (анкета)

Молодой специалист, аренда, цель — подушка **2× доход** за 12–18 месяцев; вклад после триггера; страховка после первого шока. *Может быть реализовано как отдельный шаблон или кампания.*

### 4.3. Прогрессия глав (анкета, vision)

| Глава | Тема |
|-------|------|
| 2 «Кредитное искушение» | Машина, кредит, страховка авто |
| 3 «Инвестор-новичок» | Облигации vs вклад |
| 4 «Своя крыша» | Ипотека, аренда vs покупка |
| 5 «Финансовая свобода» | Диверсификация (**без акций** в нашем scope — облигации, активы, подушка) |

### 4.4. Активный период

Микровизуализации (работа, смс банка) — vision. **Сейчас:** закрытие месяца по кнопке, карусель событий, несколько финансовых действий за сессию 1–3 мин ([`TMA_USER_FLOWS.md`](docs/foundation/TMA_USER_FLOWS.md)).

### 4.5. «Карта финансового здоровья»

🛡️ Защита | 💰 Доход | 📉 Долг | 😊 Благополучие — radar/шкалы; **цифры по тапу** (overview + analytics).

### 4.6. События-вызовы

**Шоки**, **искушения**, **возможности**; предвестники за 1–2 периода — целевое.

### 4.7. Реиграбельность

Новая игра + другой шаблон; «Уроки жизни», модификаторы — vision.

## 10. Дорожная карта разработки

*Оригинал из [`QUESTIONNAIRE.md`](QUESTIONNAIRE.md) §«Рекомендации к началу разработки». В колонке **ТВОЙ ХОД** — фактический статус и отступления.*

**Главный принцип (анкета):** всегда держать **работающий билд**; каждые 1–2 недели — расширение, которое можно пощупать.

### 10.1. Этап 0: Подготовка и выбор инструментов (1–2 дня)

**Анкета:** платформа Web; стек на выбор; локальное хранение.

**ТВОЙ ХОД:** ✅ **Web TMA** (React + Vite); **FastAPI + PostgreSQL**; JWT; не LocalStorage — серверные профили.

### 10.2. Этап 1: «Скелет» — главный игровой цикл (~1 нед.)

**Анкета:** месяц, деньги, зарплата, расходы, кнопка «следующий месяц».

**ТВОЙ ХОД:** ✅ Пошаговый период (закрытие месяца), `period_index`, балансы, `process_period_end`, снимки.

### 10.3. Этап 2: Первый инструмент — вклад (~1 нед.)

**Анкета:** *«одно действие в месяц»* + вклад с %.

**ТВОЙ ХОД:** ✅ Вклад и облигации; **❌ правило одного действия не используем** — несколько операций за период.

### 10.4. Этап 3: Первое событие и визуальная обратная связь (1–2 нед.)

**Анкета:** 1–2 события, карточка, метафора баланса.

**ТВОЙ ХОД:** ✅ 2 события/период, карусель, тосты; overview с цифрами + UI-акценты.

### 10.5. Этап 4: Система уровней и достижений (~2 нед.)

**Анкета:** достижения → XP → разблокировки; экран «Развитие».

**ТВОЙ ХОД:** ✅ character XP/level сняты; 🟡 **достижения** (движок + API, UI M12); разблокировки механик — из blueprint шаблона, не level-gates.

### 10.6. Этап 5: Прототип 1–2 уровней для плейтеста (2–3 нед.)

**Анкета:** 5–7 событий, 3/месяц, дневник; показать 3–5 людям; проверить «одно действие».

**ТВОЙ ХОД:** 🟡 Каталог событий MVP 1.1 (≥12 сидов), tier; **формальный плейтест Pre-Alpha ⬜**; проверяем **несколько действий** и перегруз событий.

### 10.7. Этап 6: Итеративное расширение

**Анкета:** уровни 3–4 (страховки, кредит), метафоры, пул событий, дневник, финальные уровни.

**ТВОЙ ХОД:** ✅ Страховки, долги, шаблоны, аналитика, **cooldown**, **Victory v2 backend**; 🟡 tier по периоду, UI целей; ⬜ дневник, Plan UI, главы-кампания.

### 10.8. Сводка: этап анкеты vs мы сейчас

| Этап анкеты | Оценка |
|-------------|--------|
| 0–3 | ✅ Пройдены (с отступлением по «одному действию») |
| 4 | 🟡 В процессе |
| 5 | 🟡 Контент богаче прототипа; плейтест не формализован |
| 6 | 🟡 Идёт итеративно |

## 11. Стратегия привлечения игроков

| Этап | N | Статус | Готово у нас (кратко) |
|------|---|--------|------------------------|
| **Pre-Alpha** | 10–20 | **📍 сейчас** | Цикл, 2 события/период, шаблоны, Victory v2 BE; 🟡 M12 UI, O1 онбординг |
| **Closed Alpha** | 50–150 | следующий | + D1/D7 гейты, плейтест баланса событий |

**Правило:** не масштабировать набор, пока Pre-Alpha не дала «хочу продолжить» без блокеров.
