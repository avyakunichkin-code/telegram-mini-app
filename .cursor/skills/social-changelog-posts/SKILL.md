# Посты по changelog (соцсети)

Скилл **только для маркетинговых артефактов**. Код, миграции, API и UI **не менять**, если пользователь явно не попросил иное.

## Быстрый старт

1. Прочитать [`docs/marketing/README.md`](../../../docs/marketing/README.md), [`POST_STYLE.md`](../../../docs/marketing/POST_STYLE.md), [`CHANGELOG_TRACKER.md`](../../../docs/marketing/CHANGELOG_TRACKER.md).
2. Собрать факты из git: `git log`, `git diff`, при необходимости — связанные `docs/vision/ideas/*.md` (без правок).
3. Сверить с трекером: не дублировать темы со статусом **Утверждено** / **Опубликовано**, если пользователь не просит переиздание.
4. Сгруппировать коммиты в **логические фичи** (одна фича может = несколько коммитов).
5. Предложить **1…N постов** (N = число законченных фич за сессию/период; мелочи объединять).
6. Черновики → `docs/marketing/posts/draft/`; после согласования → `docs/marketing/posts/` без префикса `draft_`.
7. Для каждого поста: текст + **картинка** (см. ниже) + обновить строку в трекере.

## Триггеры (фразы пользователя)

| Фраза | Действие |
|-------|----------|
| «Давай подведём итог» | Итог сессии/ветки → 1 обзорный или несколько тематических постов |
| «Напиши пост» / «Напиши посты» | 1 или N черновиков по неосвещённым фичам |
| «Давай расскажем» | Тон «история разработки», меньше техники |
| «Что нового в проекте» | С фокусом на игрока, не на стек |
| «Итоги сессии» | Только коммиты с последнего освещённого в трекере |
| «Анонс» / «материалы для публикации» | Готовые артефакты + чеклист публикации |
| «Пост в канал / телеграм» | Формат под Telegram + ссылка на [`TELEGRAM_PUBLISHING.md`](../../../docs/marketing/TELEGRAM_PUBLISHING.md) |

## Именование артефактов

Базовый slug: `YYYY-MM-DD_<краткая-тема-латиницей>`.

| Состояние | Папка | Пример |
|-----------|--------|--------|
| Черновик | `docs/marketing/posts/draft/` | `draft_2026-05-23_real-estate-catalog_post.md` |
| Утверждено | `docs/marketing/posts/<slug>/` | каталог поста |

В каталоге поста (утверждённого):

```text
<slug>/
  post.md          # текст: заголовок, тело, хештеги, CTA
  image.png        # иллюстрация для соцсети
  meta.json        # опционально: платформа, дата, commit refs
```

Связь с трекером: колонка **Артефакт** = путь к папке или `draft/...`.

## Трекер тем

Файл: [`docs/marketing/CHANGELOG_TRACKER.md`](../../../docs/marketing/CHANGELOG_TRACKER.md).

| Колонка | Кто ставит | Когда |
|---------|------------|--------|
| **Утверждено** | Агент — только после явного «утверждаем» / «ок, публикуем текст» от пользователя | Перенос из `draft/`, снятие префикса `draft_` |
| **Опубликовано** | **Только пользователь** | После реальной публикации; агент не ставит сам |

Новую тему добавлять в трекер при первом черновике; **Утверждено** — не раньше согласования.

## Текст поста

Следовать [`POST_STYLE.md`](../../../docs/marketing/POST_STYLE.md). До утверждения стиля — предлагать 1–2 варианта тона и спрашивать.

Минимум в `post.md`:

- Платформа (Telegram / VK / …) или «универсальный»
- Заголовок (опционально для TG)
- Тело (короткие абзацы, без внутренних путей репозитория)
- CTA (игра, канал, обратная связь)
- Хештеги (если уместно)

Язык по умолчанию: **русский**. Технические детали — только если усиливают доверие аудитории разработки.

## Картинки

Обязательно для каждого поста.

1. Прочитать [`docs/reference/CHARACTER_MONETKA.md`](../../../docs/reference/CHARACTER_MONETKA.md) и [`docs/reference/brandbook/BRANDBOOK.md`](../../../docs/reference/brandbook/BRANDBOOK.md).
2. Использовать **GenerateImage** с `reference_image_paths`:  
   `docs/reference/assets/monetka-mascot.png` и/или поза из `docs/reference/assets/monetka-poses/`.
3. В промпте: cartoon-монета с ручками/ножками, палитра Money Quest (тёмный фон + акцент Quest Violet / золото монеты), **без** крипто-мемов и паники, без мелкого нечитаемого текста на картинке.
4. Сохранять как `image.png` рядом с `post.md`.

Если генерация не устраивает — итерация по feedback пользователя, не переходить к **Утверждено** без согласованной картинки.

## Сбор изменений из git

```bash
git log --oneline -30
git log --format="%h %s" <since>..HEAD   # since = последний commit из meta.json или уточнить у пользователя
```

Группировка:

- Несколько коммитов → одна **тема**, если одна пользовательская ценность.
- WIP / «fix typo» / только тесты без фичи — не отдельный пост; включать в «прочее» или пропускать.
- Миграции и баланс — переводить на язык игрока («новые типы жилья», не «0029_real_estate»).

## Чего не делать (расширяемый список)

Добавлять сюда пункты, которые всплыли при согласовании постов.

- Не менять `backend/`, `frontend-react/`, миграции, `api.js`, rules, CI.
- Не коммитить и не пушить без явной просьбы.
- Не ставить **Опубликовано** и не выдавать Bot Token в чат.
- Не обещать в постах то, чего нет в коде / SPEC.
- Не публиковать в Telegram автоматически без отдельной настройки (см. TELEGRAM_PUBLISHING).
- Не дублировать уже **Утверждено** темы без запроса.

## Диалог согласования

1. Показать план: темы ← коммиты, число постов.
2. Черновики в `draft/`.
3. Дождаться правок или «утверждаем».
4. Перенести в `posts/<slug>/`, обновить трекер (**Утверждено** = да).
5. Напомнить: **Опубликовано** пользователь отметит сам после выкладки.
