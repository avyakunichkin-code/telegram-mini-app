## 1. Основные принципы бренда

**ТВОЙ ХОД** — обучающая игра про личные финансы через игровые периоды, решения и последствия. В целевой версии продукт разделяет **игровой** и **планировочный** режимы (см. [`docs/vision/ideas/money-quest-evolution-after-mvp.md`](../../vision/ideas/money-quest-evolution-after-mvp.md) §II); визуальный язык бренда единый для обоих.

| Параметр | Описание |
|----------|-----------|
| **Характер** | Уверенный, понятный, без снисходительности; «наставник, а не банковский робот». |
| **Тон голоса** | Короткие формулировки, активный залог («Получи зарплату», а не «Предполагается получение дохода»). |
| **Избегаем** | Панических формулировок, финансового жаргона без расшифровки, обещаний быстрого обогащения, обращения «Вы / Ваш / клиент». |
| **Маскот** | **Монетка** — напарница-наставница, не название продукта; только **ты / твой / тебе** (см. [`CHARACTER_MONETKA.md`](../CHARACTER_MONETKA.md)). |
| **Подпись бренда** | **Финансы как игра** — обязательна в маркетинге и на стартовых экранах с полным lockup (G1). |
| **Хештег (соцсети)** | `#ТвойХод` (не `#MoneyQuest` в новых материалах). |

Этот документ задаёт минимально достаточные правила, чтобы материалы **узнавались как ТВОЙ ХОД** на любой платформе.

### 2.3 Варианты на фонах

| Вариант | Где применять |
|---------|----------------|
| **Основной (светлый)** | Полноцветный знак на белом/`#FFFFFF` или `tg-theme-bg` в светлой теме. |
| **Тёмный инверс** | Однотонный светлый знак (белый или `#F3F4F6`) на тёмном фоне `#0F1115`, `#16171D`, градиенте палитры. |
| **Монохром** | Один канал текста без градиентов для низкой печати, факсовых форм и штемпелей. |

### 2.4 Минимальные размеры (цифровые)

| Носитель | Минимальная высота блока знака |
|----------|-------------------------------|
| TMA hero / табы (G2 compact) | **~36–44 px** по ширине макета; не сжимать ниже читаемости 3D-контуров. |
| Стартовые экраны (G1 full) | По макету `BrandMark` — обычно **ширина ~240–280 px** на 390px viewport. |
| Монограмма **ТХ** (pill) | **не ниже 24 px** высоты глифа. |
| Аватар TG / соцсети | Отдельный квадратный кроп **ТХ** или упрощённый знак — **≥ 108×108 px** содержательной части. |
| Печать визитки | **не менее 8 mm** высоты знака. |

### 2.5 Защитное поле (clear space)

Вокруг логоблока — **не менее 0,5 высоты монограммы** свободного поля без текста и графики.

### 2.6 Недопустимо

- искажение пропорций (растягивание по одной оси);  
- обводка только ради видимости вместо выбора правильного контраста;  
- чужие контейнеры (кружки произвольного цвета) без палитры;  
- добавление символов валют **внутрь** знака (₽ € $) без отдельного макета;  
- множественные градиенты на маленьких размерах (ниже минимальных);  
- размещение на «шумном» фото без затемняющего слоя минимум **40 % чёрной заливки** под знаком.

## 4. Типографика

### 4.1 Семейство шрифтов

**Различайте носитель.** В самом приложении ТВОЙ ХОД пользователь видит **шрифт Telegram-клиента**; Inter — основной интерфейсный шрифт **для стендалон-сайтов, презентаций и статичной графики вне клиента.**

| Носитель | Основной интерфейсный текст | Комментарий |
|----------|----------------------------|-------------|
| **Telegram Mini App (MQX)** | Системный стек платформы | Размеры и иерархия фиксируются `--mq-fs-body`, `--mq-fs-caption`, `--mq-fs-small` на `#root`; не подключать Inter как основной текст без явного продуктового решения |
| **Маркетинговый веб-сайт, лендинг, баннеры** | **Inter** Variable | Запас: system-ui, Segoe UI, Roboto |
| **Код, табличные дампы и метки времени в UI** | **JetBrains Mono** | ui-monospace, Consolas |

### 4.2 Иерархия (лендинг / веб без TMA стека)

Столбцы ниже — ориентир для **выхода за пределы** мини-приложения (сайты, промо-макеты). Внутри TMA сверяться с таблицей в **разделе 4.5**.

| Стиль | Размер | Вес | Межстрочный | Применение |
|-------|--------|-----|--------------|-------------|
| **H1** | 28–32 px | Semibold | 1.15 | Герой заголовки лендинга |
| **H2** | 22–24 px | Semibold | 1.2 | Секции |
| **H3** | 17–18 px | Medium | 1.25 | Подсекции, карточки |
| **Body** | 15–17 px | Regular | 1.45 | Основной текст |
| **Caption** | 12–13 px | Regular | 1.35 | Подписи, легальный текст |
| **KPI число** | 20–24 px | Tabular nums | 1 | Балансы, таймер (monospace допустим) |

### 4.3 Примеры правильного использования

- Один акцентный вес заголовка **на экран**; списки — с отступами, не узкими «столбцами» через пробелы.  
- Денежные суммы: пробел тысяч (в RU локали) **или** `1 234 567` с неразрывным пробелом; символ ₽ после числа через неразрывный пробел.  
- Выравнивание: основной текст **по левому краю** в карточках (не центрировать всё подряд в мини-приложении).

### 4.4 Примеры неправильного использования

| Неправильно | Почему |
|-------------|---------|
| Капитал всего заголовка (КРИТИЧНОЕ СОБЫТИЕ) | Крикливо и снижает доверие |
| 4+ начертания на одном экране без системы | Визуальный шум |
| Очень узкий интерлиньяж (<1.35) для body | Хуже читается в дороге |
| Мелкий текст <12 px для важной информации | Плохая доступность |

### 4.5 Telegram Mini App: единая шкала на игровом экране

Цель — чтобы карточки, списки и кнопки **читались одним взглядом**, без ощущения «собрано из разных макетов».

**Ограничение:** на одном экране допускается не больше **четырёх** визуально различимых текстовых уровней (включая числа и подписи полей). Если нужно больше смыслов — разбить на второй экран или свернуть второстепенное в подпись/раскрывающийся блок.

| Уровень | Размер в реализации | Вес | Применение |
|---------|---------------------|-----|------------|
| **1. Заголовки секций Telegram UI** (`Section header`) | задаётся библиотекой (ориентир ~15–16 px eq.) | Semibold | Имя блока: «Финансы», «Действия периода» |
| **2. Основной текст и значения KPI** | **15 px** (`--mq-fs-body`) | Regular / Semibold для сумм | Тело карточки, крупные числа в строках статистики |
| **3. Подписи полей и вторичные строки** | **12 px** (`--mq-fs-caption`) | Regular / Medium | Labels у `Input`, подсказки под заголовком секции, подписи к барам аналитики |
| **4. Микроcopy (по необходимости)** | **11 px** (`--mq-fs-small`) | Regular | Редкие служебные подсказки; использовать только если без потери смысла не обойтись |

**Правила:**

- **Шрифт:** в TMA не подключать кастомные семейства без задачи; системный стек платформы (`system-ui`, Segoe UI, Roboto…). Моноширинный — только таймер и явные «цифровые табло».  
- **Цифры:** суммы в одном стиле (`tabular-nums`, единый компонент отображения денег); проценты и коэффициенты — тот же уровень, что соседний текст строки.  
- **Акцент цветом:** брендовый **Quest Violet** для основного CTA и активных состояний нижнего меню и табов; зелёный Emerald — для «в плюс», красный — ошибки/просрочки, без смешения одного и того же типа статуса двумя цветами без подписи.

В коде приложения шкала зафиксирована в `#root` через CSS-переменные `--mq-fs-body`, `--mq-fs-caption`, `--mq-fs-small` (`frontend-react/src/index.css`).

## 5. Система интерфейса MQX (Telegram Mini App)

**MQX** — единый премиум-слой поверх `@telegram-apps/telegram-ui`: фиолетовый **hero**, стеклянные **карточки**, нижняя навигация и токены, согласованные с темой Telegram. Продукт должен восприниматься как **одно** приложение на всех вкладках игры, экранах старта и авторизации.

### 5.1 Каркас экрана

| Уровень | Класс / компонент | Назначение |
|--------|-------------------|------------|
| Внешний отступ страницы | `app-shell mq-page` (игра / поток до игры); `mq-page--auth` (логин/регистрация) | Единые поля и декоративный фон `mq-page__decor` (лёгкая сетка и градиент, уважает `prefers-reduced-motion`). |
| Колонка приложения | `#root`, `max-width: 480px` | Одна колонка TMA; не раздувать контент под десктоп. |
| Оболочка MQX | `mqx-screen` → `mqx-frame` | Рамка с мягкими радиальными бликами бренда и **фоном из темы**: `--mqx-surface-a` / `--mqx-surface-b` (не фиксированный «вечный белый» блок в тёмном клиенте). |
| Компонент-обёртка | `MqxShell` | `header` (hero) + `mqx-content` + опциональный `contentClassName` (напр. `mqx-auth`). |
| Унифицированный таб-hero | `MqxTabHero` | `BrandLogo variant="compact"` (G2), контекстная пилюля раздела, при необходимости «призрачная» пилюля справа. |
| Старт / auth | `BrandMark` → `BrandLogo variant="full"` | G1 по центру: **ТВОЙ ХОД** + tagline в alt/подписи. |
| Онбординг с Монеткой | `MqxMonetkaDialogScreen`, `MonetkaBubbleScreen` | Монетка + реплика; лого по макету экрана (см. §7.4). |

**Главная вкладка** — **`mqx-hero--compact`**: G2 слева, таймер и прогресс периода, «Играть / Пауза». Остальные вкладки — **`mqx-hero mqx-hero--tab`** с тем же compact; «Финансы» — **`mqx-hero--capital`**. Стартовые экраны — G1 по центру, прозрачный PNG, без подложки-карточки под lockup.

### 5.2 Цвет и тема в рантайме

**Приоритет в TMA:** `var(--tg-theme-bg-color)`, `var(--tg-theme-text-color)`, `var(--tg-theme-hint-color)`, `var(--tg-theme-secondary-bg-color)` для фона, текста, бордеров и приглушённых подписей. Это задаёт клиент Telegram при открытии WebApp (`syncTelegramThemeToRoot` дописывает hex в инлайновые свойства `#root`, при событии смены темы — повторная синхронизация).

**Брендовый акцент** поверх темы клиента не отдаём «голубому» акценту Telegram по умолчанию для наших основных действий:

| Переменная | Роль |
|-------------|------|
| `--mq-accent-fill` / переопределение `--tgui--button_color` | Заливка primary CTA и активного состояния значимых контролов. |
| `--tgui--link_color`, `--tgui--accent_text_color`, `--tgui--toast_accent_color` | Ссылки, акцентный текст и тосты в фирменном фиолетовом ряду. |
| `--tgui--outline` | Обводка контролов telegram-ui смешением брендового violet и `--tg-theme-hint-color`. |

**Семантика данных в интерфейсе (не смешивать без подписи):**

| Смысл | Брендовый якорь | Типичное применение в MQX |
|-------|-----------------|---------------------------|
| Плюс к деньгам, успех действия | Signal Emerald `#059669` | Подушка, позитивные бары аналитики, «да» на событиях. |
| Обязательства, риск, отрицательный поток | Danger `#DC2626` и производные soft | Просрочка, ошибки платежей, отрицательные ряды. |
| Ожидание, время | Warning Amber `#D97706` | Дополнительное внимание к таймингу (не дублировать danger). |

### 5.3 Поверхности MQX («стекло» и тени)

Переменные на `#root` (fallback на белый только вне клиента Telegram):

| Токен | Назначение |
|-------|------------|
| `--mqx-ink` | Текст премиум-блоков: смесь от `--tg-theme-text-color`. |
| `--mqx-glass-strong` … `--mqx-glass-softer` | Полупрозрачные заливки карточек и строк поверх темы (`color-mix` с `--tg-theme-bg-color`). |
| `--mqx-shadow-soft` | Мягкая тень рамы `.mqx-frame`. |
| `--mqx-shadow-card` | Карточные тени блоков финансов/аналитики. |

**Правило:** не вводить новые полноэкранные фиолетовые градиенты на каждую секцию; **один сильный** герой на экран (или вкладку) достаточен. Дополнительные акценты — через маленькие градиенты на KPI-чипах и иконках `.mqx-mini__icon`.

### 5.4 Компонентная иерархия (ориентиры по именам классов)

| Паттерн | Классы | Когда использовать |
|---------|--------|---------------------|
| Карточка цели | `mqx-card mqx-card--goal` | Прогресс к победе MVP на главной. |
| Карточка раздела | `mqx-card`, `mqx-card__kicker`, `mqx-card__title`, `mqx-card__sub` | Короткий кикер uppercase, один заголовок, подзаговор на 1–2 строки. |
| Сетка KPI | `mqx-grid2` + `mqx-mini` + `mqx-accent--violet` / emerald / sky / amber | Четыре квадратика на главной; иконки 24 px outline по духу раздела 7.2. |

Сегмент **«Добавить \| Мои (N)»** — внутри карточки активного раздела, не в hero. Макет: [`design-lab/capital-page/`](../../../design-lab/capital-page/). |

| Нижняя навигация | `.bottom-nav`, `.tma-nav-icon-btn`, высота `--tma-tabbar-height: 56px` | Не дублировать второй таббар на экранах. |

### 5.5 Скругления и плотность (цифровой UI)

Ориентиры из текущего CSS (не размножать произвольные значения без причины):

| Элемент | Ориентир radius |
|---------|----------------|
| Рама приложения `.mqx-frame` | ~18 px |
| Крупные карточки портфеля / уровень аналитики | ~22 px |
| Кнопки hero `.mqx-btn`, чипы | ~14 px |
| Мелкие контролы, икон-кнопки | ~10–14 px |

**Межстрочный** основного текста в колонке: **1.42** при `--mq-fs-body: 15px`.

### 5.6 Модальные окна и события

| Ситуация | Паттерн |
|----------|---------|
| Модалка внутри telegram-ui `Modal` | Обёртка `.mqx-modal` + внутри `.mqx-card` с заголовком и действиями `mq-modal-actions`. |
| Оверлей событий периода | `role="dialog"`, `aria-modal="true"`, фирменная панель `.mqx-events-*`, закрытие явной кнопкой; свайпы и точки карусели как дополнение к фокусу клавиатуры. |

Копирайт в модалках: **объяснить последствие** («сгорит зарплата»), не ограничиваться одним словом «Внимание».

### 5.7 Движение и доступность

- **Микровзаимодействия:** короткий `transform`/`opacity` на кнопках в колонке `#root` там, где не отключены исключения (иконки таббара, точки карусели, выбор событий).  
- **`prefers-reduced-motion`: уважать** — отключать декор `mq-page__decor`, вводные анимации `mq-stack-animate`, слайд карусели событий.  
- **Фокус:** `focus-visible` с фиолетовым контуром на кнопках; таббар с `aria-label` и `aria-current="page"`.  
- **Скрытые подписи:** класс `.visually-hidden`, если визуально дублируется заголовок экрана.

### 5.8 Голос интерфейса (RU)

Согласуйте с разделом 1 и со [`SPEC_FRONTEND_UI`](../../specs/SPEC_FRONTEND_UI.md):

- Одна главная формулировка действия на экран там, где это уместно.  
- Подписи полей короткие; единицы явно («₽», «сек», «Период N»).  
- Без панических заголовков и без обещания быстрого обогащения.  
- Англоязычные хвосты в production UI недопустимы (названия API и slug шаблонов — только вторичными строками или во внутренних логах).

## 6. Макеты и применения

### 6.1 Визитка (90 × 50 mm, пример содержания)

**Лицевая:** полный lockup **G1** (ТВОЙ ХОД + «Финансы как игра») или монограмма **ТХ** на узком носителе.  
**Обратная:** имя контакта · роль · email · ссылка `t.me/...` или QR на бота · цвет акцента только полосой 4 мм снизу.

### 6.2 Квадрат соцсетей (1080 × 1080 px)

Структура: верхушка **G1/G2** или **ТХ**; середина один **message** («Играй в месяц за 10 минут»); низ палитрой **Line**. Текста не более 25% квадрата. Монетка — опционально, не перекрывает lockup.

### 6.3 Сторис / вертикаль (1080 × 1920)

Безопасная зона: отступ сверху/снизу **140 px**. CTA блок в нижней трети, контраст с фото ≥ 4.5:1.

### 6.4 Горизонтальный баннер (например, 1200 × 628)

Лого слева в треть; текст и CTA центр/право на однотонной подложке **Mist** или градиенте из раздела 3. Не более **одного градиента** и одного ключевого CTA.

## 7. Фирменные дополнительные элементы

### 7.1 Паттерн

**«Сетка периода»:** тонкая сетка 8 px, цвет `#E5E7EB`, непрозрачность **20–35%**. Использовать как фон **не под мелкий текст** — только за hero или в промо-блоках с крупными заголовками.

### 7.2 Иконсет (направление)

Стиль **24 px outline**, округление углов 1.5 px, толщина штриха **1.75 px**. Тематика: календарь/период, кошелёк, подушка щитом, минимальный график. Не смешивать filled и outline в одной строке навигации.

### 7.3 Иллюстрации

Плоские фигуры, **немного градиента** только на объектах-«наградах». Избегать реалистичных банковских зданий; предпочтение абстрактным «целям», «дорожке», «решениям». Герой-маркетинг — сцена и **Монетка** из постов; в UI приложения сцена **не** входит в lockup логотипа.

### 7.4 Персонаж Монетка (маскот)

Канон: [`CHARACTER_MONETKA.md`](../CHARACTER_MONETKA.md). Кратко для визуала:

| Параметр | Правило |
|----------|---------|
| **Роль** | Напарница в онбординге (Mission Brief), coach marks; **не** комментирует каждое игровое событие. |
| **Форма** | Круглая монета, cartoon; PNG с прозрачным фоном. |
| **Дефолт** | `docs/reference/assets/monetka-mascot.png` → prod `frontend-react/src/assets/monetka-mascot.png` |

| **Композиция** | Пузырь реплики — **CSS**, не запечён в PNG; `sit-edge` — низ монеты на кромке `.mqx-monetka-dialog__speech`. |
| **Нельзя** | ₽ на лице; CSS-заглушка вместо ассета; «Вы / Ваш» в копирайте Монетки. |

## 9. Чеклист поддержки идентичности перед публикацией

- Контраст текста WCAG AA где возможно.  
- Одна главная кнопка на экран **вне** telegram-ui там, где много вторичных ссылок; внутри TMA главный CTA в hero или в ключевой карточке периода.  
- Логотип — только **G1/G2 PNG** (или согласованный экспорт из них); нет плоского SVG и **MQ** / Money Quest.  
- Нет запрещённых вариаций знака из раздела 2.6.  
- **Монетка:** PNG с alpha; обращение на **ты**; поза из каталога, не новая «от руки» без записи в `CHARACTER_MONETKA.md`.  
- Суммы и периоды подписаны единицами («₽», «Период 7», «сек» у таймера).  
- **MQX:** новый экран или вкладка — через `MqxShell`/`MqxTabHero` или документированное исключение; не более одного доминирующего фиолетового геро-блока.  
- **Темизация:** блоки MQX используют `--mqx-glass-*` и `--tg-theme-*`, а не голый белый там, где ожидается тёмный клиент Telegram.  
- **Движение:** при включённом «уменьшить движение» системы не должны выпадать ключевые смыслы (карусель событий остаётся доступной без анимации смены карточек).  
- **Модальные предупреждения:** указано действие последствий, не только общее «осторожно».
