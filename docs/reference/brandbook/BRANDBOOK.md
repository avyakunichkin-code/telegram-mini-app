# Брендбук Money Quest

**Версия документа:** 1.0  
**Дата:** май 2026  
**Применение:** Telegram Mini App, веб-лендинг, социальные сети, печать, партнёрские материалы

---

## 1. Основные принципы бренда

**Money Quest** — обучающая игра про личные финансы через игровые периоды, решения и последствия. В целевой версии продукт разделяет **игровой** и **планировочный** режимы (см. [`docs/vision/ideas/money-quest-evolution-after-mvp.md`](../../vision/ideas/money-quest-evolution-after-mvp.md) §II); визуальный язык бренда единый для обоих.

| Параметр | Описание |
|----------|-----------|
| **Характер** | Уверенный, понятный, без снисходительности; «наставник, а не банковский робот». |
| **Тон голоса** | Короткие формулировки, активный залог («Получи зарплату», а не «Предполагается получение дохода»). |
| **Избегаем** | Панических формулировок, финансового жаргона без расшифровки, обещаний быстрого обогащения. |

Этот документ задаёт минимально достаточные правила, чтобы материалы **узнавались как Money Quest** на любой платформе.

## 2. Логотип и знак

### 2.1 Состав знака

- **Фирменное сокращение:** **MQ** (монограмма)  
- **Словесное название:** **Money Quest** (капитализируются оба слова; без «moneyquest» без пробела).

### 2.2 Конструктор (ориентир)

| Элемент | Правило |
|---------|---------|
| **Монограмма** | Строчные связки или «монолит»: буквы M и Q узнаваемы; внутренние пробелы — не ниже нижнего профиля. |
| **Словесное написание** | Гегевой **Money** + начертание того же семейства или пары для **Quest** (допускается лёгкий контраст веса: Medium + Semibold). |
| **Подпись (tagline)** | Опционально: «Финансы как игра». Не крупнее 40% высоты логоблока без отдельного согласования. |

Файловые ресурсы: при появлении финальных файлов добавьте раздел приложения со ссылкой на архив SVG/PDF для печати. До выпуска вектора используйте **спецификацию цветов** и **минимальный защитный элемент** ниже.

### 2.3 Варианты на фонах

| Вариант | Где применять |
|---------|----------------|
| **Основной (светлый)** | Полноцветный знак на белом/`#FFFFFF` или `tg-theme-bg` в светлой теме. |
| **Тёмный инверс** | Однотонный светлый знак (белый или `#F3F4F6`) на тёмном фоне `#0F1115`, `#16171D`, градиенте палитры. |
| **Монохром** | Один канал текста без градиентов для низкой печати, факсовых форм и штемпелей. |

### 2.4 Минимальные размеры (цифровые)

| Носитель | Минимальная высота блока знака |
|----------|-------------------------------|
| Мобильный UI (TMA header) | **24 px** (высота монограммы). |
| Социальные сети аватар | **не менее 108×108 px** содержательной части знака во кропе (не обрезать контуры). |
| Печать визитки | **не менее 8 mm** высоты монограммы. |

### 2.5 Защитное поле (clear space)

Вокруг логоблока — **не менее 0,5 высоты монограммы** свободного поля без текста и графики.

### 2.6 Недопустимо

- искажение пропорций (растягивание по одной оси);  
- обводка только ради видимости вместо выбора правильного контраста;  
- чужие контейнеры (кружки произвольного цвета) без палитры;  
- добавление символов валют **внутрь** знака (₽ € $) без отдельного макета;  
- множественные градиенты на маленьких размерах (ниже минимальных);  
- размещение на «шумном» фото без затемняющего слоя минимум **40 % чёрной заливки** под знаком.

## 3. Цветовая палитра и сочетания

### 3.1 Основные цвета (бренд)

| Имя | HEX | Применение |
|-----|-----|------------|
| **Quest Violet** | `#6D28D9` | Основной акцент CTA (когда не противоречит Telegram theme). |
| **Quest Violet Deep** | `#5B21B6` | Hover/актив акцента на вебе. |
| **Signal Emerald** | `#059669` | Деньги «в плюс», подушка, успех действия. |
| **Ink** | `#0F1115` | Заголовки и первичный текст вне Telegram theme. |

### 3.2 Вторичные и нейтральные

| Имя | HEX | Применение |
|-----|-----|------------|
| **Mist** | `#F5F6F8` | Фон секций карточек (веб). |
| **Slate** | `#6B7280` | Подписи, вторичный текст. |
| **Line** | `#E5E7EB` | Разделители, карточные границы. |

### 3.3 Системные статусы (игра и UI)

| Имя | HEX | Применение |
|-----|-----|------------|
| **Warning Amber** | `#D97706` | Таймер, «осталось мало времени». |
| **Danger** | `#DC2626` | Просрочка, ошибка платежа, game over UX. |

### 3.4 Тёмная тема (веб без TG)

Фон **`#16171D`**, текст основной **`#F3F4F6`**, акцент — **Quest Violet** или осветление до **`#8B5CF6`** для достаточного контраста (WCAG 4.5:1 против фона там, где возможно).

### 3.5 В Telegram Mini App

**Приоритет:** переменные `tg-theme-*` Telegram для фона и основного текста. Основные **кнопки действия и вкладки внутри Money Quest** унифицируются **Quest Violet**, чтобы нижнее меню, табы «Финансы» и CTA были визуально согласованы (не «голубые кнопки + фиолетовый бар»). Детали см. §4.5.

### 3.6 Примеры допустимых сочетаний

**A.** Белый фон + текст Ink + акцент Quest Violet для кнопки.  
**B.** Тёмный фон `#0F1115` + текст `#F9FAFB` + акцент emerald для позитивных метрик.  
**C.** Светло-серый Mist + тёмная типографика + тонкая линия Line; акцент только на одном действии на экран.  
**D.** Карточка с градиентом **Violet Deep → Indigo `#4338CA`** — только для **hero** и ключевых промо-баннеров, не для плотного текста под ним без подложки.

**Избегать:** фиолетовый текст на фиолетовом без достаточного контраста; красный и зелёный рядом для одних и тех же типов сообщений без подписей (дальтоника).

## 4. Типографика

### 4.1 Семейство шрифтов

| Категория | Приоритет 1 | Запас |
|-----------|-------------|-------|
| **Интерфейс и веб** | **Inter** Variable | system-ui, Segoe UI, Roboto |
| **Код и цифры в таблице** | **JetBrains Mono** | ui-monospace, Consolas |
| **TMA внутри Telegram** | По умолчанию **системный стек платформы** | не переопределять без нужды |

### 4.2 Иерархия (цифровой веб/TMA текстовые блоки)

| Стиль | Размер | Вес | Межстрочный | Применение |
|-------|--------|-----|--------------|-------------|
| **H1** | 28–32 px | Semibold | 1.15 | Герой заголовки лендинга |
| **H2** | 22–24 px | Semibold | 1.2 | Секции |
| **H3** | 17–18 px | Medium | 1.25 | Подсекции, карточки |
| **Body** | 15–17 px | Regular | 1.45 | Основной текст |
| **Caption** | 12–13 px | Regular | 1.35 | Подписи, легальный текст |
| **KPI число** | 20–24 px | Tabular nums | 1 | Балансы, таймер (monospace допустим) |

### 4.3 Примеры правильного использования

- Один акцентный вес заголовка **на экран**; списки — с отступами, не узкими «столбцами» через пробелы.  
- Денежные суммы: пробел тысяч (в RU локали) **или** `1 234 567` с неразрывным пробелом; символ ₽ после числа через неразрывный пробел.  
- Выравнивание: основной текст **по левому краю** в карточках (не центрировать всё подряд в мини-приложении).

### 4.4 Примеры неправильного использования

| Неправильно | Почему |
|-------------|---------|
| Капитал всего заголовка (КРИТИЧНОЕ СОБЫТИЕ) | Крикливо и снижает доверие |
| 4+ начертания на одном экране без системы | Визуальный шум |
| Очень узкий интерлиньяж (<1.35) для body | Хуже читается в дороге |
| Мелкий текст <12 px для важной информации | Плохая доступность |

### 4.5 Telegram Mini App: единая шкала на игровом экране

Цель — чтобы карточки, списки и кнопки **читались одним взглядом**, без ощущения «собрано из разных макетов».

**Ограничение:** на одном экране допускается не больше **четырёх** визуально различимых текстовых уровней (включая числа и подписи полей). Если нужно больше смыслов — разбить на второй экран или свернуть второстепенное в подпись/раскрывающийся блок.

| Уровень | Размер в реализации | Вес | Применение |
|---------|---------------------|-----|------------|
| **1. Заголовки секций Telegram UI** (`Section header`) | задаётся библиотекой (ориентир ~15–16 px eq.) | Semibold | Имя блока: «Финансы», «Действия периода» |
| **2. Основной текст и значения KPI** | **15 px** (`--mq-fs-body`) | Regular / Semibold для сумм | Тело карточки, крупные числа в строках статистики |
| **3. Подписи полей и вторичные строки** | **12 px** (`--mq-fs-caption`) | Regular / Medium | Labels у `Input`, подсказки под заголовком секции, подписи к барам аналитики |
| **4. Микроcopy (по необходимости)** | **11 px** (`--mq-fs-small`) | Regular | Редкие служебные подсказки; использовать только если без потери смысла не обойтись |

**Правила:**

- **Шрифт:** в TMA не подключать кастомные семейства без задачи; системный стек платформы (`system-ui`, Segoe UI, Roboto…). Моноширинный — только таймер и явные «цифровые табло».  
- **Цифры:** суммы в одном стиле (`tabular-nums`, единый компонент отображения денег); проценты и коэффициенты — тот же уровень, что соседний текст строки.  
- **Акцент цветом:** брендовый **Quest Violet** для основного CTA и активных состояний нижнего меню и табов; зелёный Emerald — для «в плюс», красный — ошибки/просрочки, без смешения одного и того же типа статуса двумя цветами без подписи.

В коде приложения шкала зафиксирована в `#root` через CSS-переменные `--mq-fs-body`, `--mq-fs-caption`, `--mq-fs-small` (`frontend-react/src/index.css`).

## 5. Макеты и применения

### 5.1 Визитка (90 × 50 mm, пример содержания)

**Лицевая:** монограмма MQ + **Money Quest** + tagline одной строкой.  
**Обратная:** имя контакта · роль · email · ссылка `t.me/...` или QR на бота · цвет акцента только полосой 4 мм снизу.

### 5.2 Квадрат соцсетей (1080 × 1080 px)

Структура: верхушка **MQ** или полный лого; середина один **message** («Играй в месяц за 10 минут»); низ палитрой **Line**. Текста не более 25% квадрата.

### 5.3 Сторис / вертикаль (1080 × 1920)

Безопасная зона: отступ сверху/снизу **140 px**. CTA блок в нижней трети, контраст с фото ≥ 4.5:1.

### 5.4 Горизонтальный баннер (например, 1200 × 628)

Лого слева в треть; текст и CTA центр/право на однотонной подложке **Mist** или градиенте из раздела 3. Не более **одного градиента** и одного ключевого CTA.

## 6. Фирменные дополнительные элементы

### 6.1 Паттерн

**«Сетка периода»:** тонкая сетка 8 px, цвет `#E5E7EB`, непрозрачность **20–35%**. Использовать как фон **не под мелкий текст** — только за hero или в промо-блоках с крупными заголовками.

### 6.2 Иконсет (направление)

Стиль **24 px outline**, округление углов 1.5 px, толщина штриха **1.75 px**. Тематика: календарь/период, кошелёк, подушка щитом, минимальный график. Не смешивать filled и outline в одной строке навигации.

### 6.3 Иллюстрации

Плоские фигуры, **немного градиента** только на объектах-«наградах». Избегать реалистичных банковских зданий; предпочтение абстрактным «целям», «дорожке», «решениям».

## 7. Кросс-платформенные гайдлайны

| Платформа | Правило |
|-----------|---------|
| **Telegram Mini App** | Следовать теме клиента для фона и текста; брендовые акценты — в собственных виджетах и иконках. |
| **Веб-сайт** | Палитра Quest Violet + Emerald; Inter; единые радиусы карточек **12 px**. |
| **Социальные сети** | Лого и цвет блоков — из палитры; не инвертировать цвет канала платформы (например, не перекрашивать ленту Threads в violet целиком). |
| **Печать** | Pantone/AP не задаются в v1 — использовать CMYK симуляцию через профиль офсета; сохранять **чистые зоны** минимальных размеров. |
| **Партнёры** | Лого не ближе **8 mm** до сторонней марки; наш знак не менее **80%** высоты их знака если в одну строку. |

## 8. Чеклист поддержки идентичности перед публикацией

- Контраст текста WCAG AA где возможно.  
- Одна главная кнопка на экран (где применимо).  
- Нет запрещённых вариаций знака из раздела 2.6.  
- Суммы и периоды подписаны единицами («₽», «Период 7»).

## 9. Экспорт в PDF

Источник этого текста:** `docs/reference/brandbook/BRANDBOOK.md`  
**Готовый к печати HTML:** `docs/reference/brandbook/brandbook-print.html`  

### Вариант A — браузер (быстро)

1. Открыть файл `brandbook-print.html` в Chrome / Edge.  
2. Печать **Ctrl+P** → **«Сохранить как PDF»**.  
3. Включить **«Фоновая графика»** если нужны образцы цвета.

### Вариант B — Pandoc

```bash
pandoc docs/reference/brandbook/BRANDBOOK.md -o MoneyQuest-Brandbook.pdf --pdf-engine=xelatex -V geometry:margin=20mm -V fontsize=11pt
```

При отсутствии LaTeX: `pandoc BRANDBOOK.md -o brandbook.docx`, затем экспорт из Word в PDF.

### Вариант C — экспорт из Cursor / VS Code

Установите расширение **Markdown PDF** → открыть `BRANDBOOK.md` → **Markdown PDF: Export (pdf)** из палеты команд.
