# Плейтест / QA — guide

Маршрут для **участников Pre-Alpha** и модераторов волны. Технические пути к коду здесь **не нужны**.

---

## С чего начать (5 минут)

1. **[`PLAYER_EXPERIENCE.md`](../PLAYER_EXPERIENCE.md)** — как играть, цель сессии, два поражения  
2. **[`PRE_ALPHA_PLAYTEST_PROTOCOL.md`](../../foundation/PRE_ALPHA_PLAYTEST_PROTOCOL.md)** — длительность, опрос, классификация багов  
3. При застревании — **[`GLOSSARY.md`](../../foundation/GLOSSARY.md)** (термины)

---

## Кого мы зовём

| | |
|--|--|
| **Размер волны** | 10–20 человек |
| **Профиль** | **30+**, готовность к сырой версии; mix «играют / не играют» приветствуется |
| **Канал** | Telegram **или** браузер **или** PWA — по ссылке из приглашения |
| **Публичный анонс** | нет |

---

## Чеклист участника за сессию

- [ ] Новая партия: **Игра** → **«Студент»** (если не указано иное)  
- [ ] Забрать **зарплату** хотя бы в одном периоде  
- [ ] Пройти **2 события** в каком-то месяце  
- [ ] Открыть **Потребности** (если блок на главном)  
- [ ] Закрыть **3–4 периода** («Закрыть месяц»)  
- [ ] Понять, **почему** проиграли или что мешает победе (если дошли)  
- [ ] Заполнить **опрос** после сессии

## Два вида поражения

| Тип | Условие |
|-----|---------|
| **Деньги** | 3 месяца подряд отрицательный счёт после закрытия |
| **Потребности** | Любая шкала на **0** три месяца подряд |

Победа — отдельно (цели на экране, обычно не раньше 7-го месяца).

---

## Как помечать фидбек

| Метка | Когда |
|-------|--------|
| **BLOCKER** | нельзя продолжить |
| **RESUME** | после блокировки экрана цифры/период не сходятся |
| **UX** | можно играть, но непонятно |
| **BALANCE** | «несправедливо», одна кнопка всегда выгоднее, одна история снова и снова — см. [`EVENTS.md`](../EVENTS.md) |
| **IDEA** | на будущее |

## KPI волны (команда, не участник)

Канон: [`KPI_AND_PHASES.md`](../KPI_AND_PHASES.md) — gate **PA-G1..G3**, target **PA-T1** (≥50% закрыли **≥5** периодов), **PA-T1s** (≥25% с **≥8**).

## Что не в фокусе этой волны
- NPS, D1/D7, конкуренты  
- **Продажа услуг финсоветника** — не предлагать участникам Pre-Alpha (воронка — [`ADVISOR_FUNNEL_AUDIENCE.md`](../ADVISOR_FUNNEL_AUDIENCE.md), после CustDev)

**Для команды (не для участника):** страхи и формулировки из advisor-doc полезны при разборе качественного фидбека и будущем опросе после 4-го периода.
