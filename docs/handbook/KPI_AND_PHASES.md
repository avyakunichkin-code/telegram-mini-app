# KPI по фазам продукта (лайтовый набор)

Числовые ориентиры для **решений go / no-go**. Не заменяют здравый смысл и качественный разбор фидбека.

**Подход:** GQM (цель фазы → вопрос → метрика), **gate** (обязательно) vs **target** (хорошо бы), **guardrail** (не ухудшать).  
**Метод:** опрос + ручной учёт + минимальная аналитика на Closed Alpha.

Связано: [`PRE_ALPHA_PLAYTEST_PROTOCOL.md`](../foundation/PRE_ALPHA_PLAYTEST_PROTOCOL.md) · [`GAME.md`](GAME.md#воронка-привлечения-игроков) · [`FEATURE_STATUS.md`](FEATURE_STATUS.md)

---

## Сводка по фазам

| Фаза | N (ориентир) | North Star (лайт) | Retention в фокусе? |
|------|--------------|-------------------|---------------------|
| **Pre-Alpha** | 10–20 | Нет блокеров + «можно звать ещё людей без сопровождения» | **Нет** (D1/D7 не меряем) |
| **Closed Alpha** | 50–150 | **D7** среди игравших хотя бы 1 период | **Да** (D1 — смотрим, не gate) |
| **Soft Launch** | 500–2k | TBD после Closed Alpha | D7/D30 |

Воронка **игра → финсоветник** — отдельные KPI: [`ADVISOR_FUNNEL_AUDIENCE.md`](ADVISOR_FUNNEL_AUDIENCE.md) (после стабильного core loop).

## Pre-Alpha (лайт)

**Цель фазы:** core loop **играбелен и понятен** без постоянного сопровождения; не доказываем retention и не полируем «магазин».

### Gate (обязательно для «go» на Closed Alpha)

| ID | Критерий | Порог | Как мерить |
|----|----------|-------|------------|
| **PA-G1** | Нет **открытых BLOCKER** | **0** незакрытых на момент решения | §8 протокола, чат |
| **PA-G2** | Команда согласна: **можно расширять круг** | да / нет (retro 30 мин) | владелец продукта + 1 dev |
| **PA-G3** | Опрос: «нельзя было продолжить без помощи» (Q6) | **≤20%** ответили «да» | форма после сессии |

Если PA-G1 fail — **no-go** до фикса. PA-G2 и PA-G3 — при пограничном «да» допустима **вторая микро-волна** (5–8 человек), не полный стоп.

### Target (лайт, не блокируют)

| ID | Критерий | Порог | Комментарий |
|----|----------|-------|-------------|
| **PA-T1** | Дошли до **≥3-го периода** | **≥60%** ответивших в опросе | Q9 опроса; stretch **70%**; не статзначимость при n≈15 |
| **PA-T2** | Понятность периода (Q1) | среднее **≥3.5** / 5 | |
| **PA-T3** | Понятность зарплаты (Q2) | **≥50%** «да» или «частично» | |
| **PA-T4** | Понимание подушки (Q3) | **≥50%** «да» или «частично» | |
| **PA-T5** | «Хотел бы ещё 1–2 периода» (Q7) | среднее **≥3.0** / 5 | намерение, не обязательство |

### Guardrail (лайт)

| ID | Критерий | Порог |
|----|----------|-------|
| **PA-R1** | Критичные **RESUME** (баланс/период не сходится) | нет повторяющегося паттерна без тикета |
| **PA-R2** | Жалобы «цифры врут» без объяснения | **0** без разбора |

### Намеренно не KPI Pre-Alpha

- D1 / D7 / NPS  
- Баланс событий по статистике  
- Конверсия в советника  
- Сравнение с «индустрией»

## Closed Alpha (лайт)

**Цель фазы:** есть сигнал, что **часть людей возвращается**; core loop держит **без массовых блокеров**.

**Когорта для retention (лайт):** все, кто **закрыл ≥1 период** в первые 7 дней после регистрации / первого входа.

### Gate (лайт)

| ID | Критерий | Порог |
|----|----------|-------|
| **CA-G1** | Открытые **BLOCKER** в support | нет системных (одиночные — ok с фиксом) |
| **CA-G2** | **D7 retention** (когорта выше) | **≥8%** |

При CA-G2 <8% — не «убить продукт», а **план**: онбординг O1, копирайт, 2-я волна Alpha; Soft Launch не открывать без retro.

### Target (лайт)

| ID | Критерий | Порог |
|----|----------|-------|
| **CA-T1** | **D1 retention** | **≥15%** (смотрим, не gate) |
| **CA-T2** | **D7 retention** | **≥10%** (хорошо); **≥12%** (отлично для niche) |
| **CA-T3** | До **period_index ≥ 4** за 7 дней | **≥30%** новых игроков |
| **CA-T4** | Median **закрытых периодов** за 7 дней на игрока | **≥4** |
| **CA-T5** | Game over в первые **5** периодов | **≤45%** партий (мягко) |

### Guardrail (лайт)

- Белый экран / невозможность войти: единичные кейсы, не волна.  
- Рост **RESUME**-тикетов неделя к неделе — стоп и фикс PW1.

### Аналитика (минимум, ⬜ в prod)

События для Closed Alpha: `period_closed`, `game_over` (+ reason), `salary_claimed`, `event_chosen`, `session_start`.  
Достаточно **таблицы когорт** (день регистрации × returned D1/D7), не полный дашборд.

## Decision tree (лайт)

```text
Pre-Alpha волна
  → PA-G1..G3 pass?
       нет → фикс блокеров → микро-волна или повтор
       да  → PA-T1..T5 как линза (не veto)
            → go Closed Alpha (50–150)

Closed Alpha 2–4 недели
  → CA-G2 D7 ≥8%?
       нет → план улучшений, не Soft Launch
       да  → CA-T2..T5 как линза
            → обсуждение Soft Launch + ужесточение KPI v2
```

---

## Ужесточение (после данных)

После **2 волн** или **≥80 ответов** в опросе пересмотреть:

| Метрика | Сейчас (лайт) | Куда можно сдвинуть |
|---------|---------------|---------------------|
| PA-T1 (≥3 период) | 60% | 70% gate |
| CA-G2 D7 | 8% | 10–12% gate |
| CA-T3 (до P4) | 30% | 40% |

Запись в [`DOC_SYNC_LOG.md`](../foundation/DOC_SYNC_LOG.md) при смене порогов.
