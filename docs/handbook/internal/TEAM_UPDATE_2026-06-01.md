Коротко о том, что уже в ветке разработки и что стоит проверить при смоуке и плейтесте. Тон — для коллег, не для канала игроков.

## Вкладка «Капитал» — структура Details | Actions v2

**Зачем:** раньше на одном экране смешивались «посмотреть, что есть» и «оформить новое» — длинные аккордеоны, формы внутри списков, тяжело на телефоне. Макет прошёл design-lab [`details-actions-round`](../../../design-lab/capital-page/details-actions-round/) и перенесён в prod.

**Как теперь устроено:**

1. Сверху по-прежнему **Доходы** и **Расходы** (разбивка периода). В заголовке — сумма (**meta M7**).
2. Ниже — переключатель **«Детали | Действия»** (визуальный разделитель).
3. **Детали** — только то, что уже оформлено: позиции с компактными кнопками «Закрыть» / «Продать» / «Отменить». Счётчики в заголовках секций — **icon + число (M8)**; у обязательств — бейдж «N долг(ов)» с красноватым tint (**M5**).
4. **Действия** — сетка плиток (3 в ряд, на узком экране 2): Депозит, Облигации, Недвижимость, Авто, Страховки, **Ипотека**, **Кредит**. Оформление открывается в **нижнем sheet**, а не внутри аккордеона.

**Что убрали с экрана (намеренно):**

- Блок-подсказка с Монеткой на вкладке «Капитал».
- Строка «В разделах ниже доступны действия…».
- Сегменты «Оформить | Мои» внутри каждого аккордеона — каталог и формы переехали в sheets.

**Ипотека и кредит** в действиях разделены: ипотека — только шаблон `mortgage`; «Кредит» — потребительский, автокредит, кредитная карта. Недвижимость и авто фильтруются по типу шаблона актива.

**Документация:** [`docs/ux/screens/finance.md`](../../ux/screens/finance.md), [`SPEC_FRONTEND_UI.md`](../../specs/SPEC_FRONTEND_UI.md) § Капитал, lab [`details-actions-round/README.md`](../../../design-lab/capital-page/details-actions-round/README.md).

## Design-lab: что починили по пути

| Проблема | Решение |
|---------|---------|
| На странице раунда «поехали» стили (битый `lab-base.css`, дубли `body`, чужие стили из старых сборок) | Пересборка через `sync-lab.sh`; в коммит только актуальный `lab-base.css` |
| Зависимость от PowerShell для sync в `details-actions-round` | **`sync-lab.ps1` удалён**, канон — **`sync-lab.sh`** (bash / Git Bash); fallback — `npm run design-lab:sync-round` |
| Действия столбиком — неудобно на TMA | Сетка **2–3 плитки в ряд** (`cap-act-list--wrap`) |
| Ипотека и кредит сливались в один пункт | Отдельные плитки и sheets |
| Meta в заголовках — текст «2 поз.» | Утверждены **M7** (потоки), **M8** (позиции), **M5** (долги) |

## Другие заметные доработки (не только финансы)

### Победа и темп игры

- Снят **жёсткий порог по номеру периода** для победы: победа определяется **целями шаблона**, без отдельного «дождитесь периода N».
- Обновлены ожидания по длительности сессии и KPI pre-alpha в [`TARGET_PLAYER_AND_SESSION.md`](../../foundation/TARGET_PLAYER_AND_SESSION.md) и связанных handbook-доках.

### Закрытие периода и подсказки

- В разборе закрытия периода (API / guidance) — поля **оплачено / не оплачено / к оплате**, учёт **новой просрочки** (`overdue_added`).
- В теле подсказки (beat) при нужных условиях показывается **превью закрытия периода** — игрок видит последствия до нажатия «Закрыть месяц».

### Онбординг O2 (guidance)

- В prod: полоса подсказок на главной, прогресс guidance в профиле, chip подушки **«ФИН.ПОДУШКА · N%»**.
- Спека: [`SPEC_onboarding-o2.md`](../../specs/features/SPEC_onboarding-o2.md).

### Контент и баланс событий

- Ребаланс каталога MVP11 (lifecycle, trade-off), усилены проверки **Pareto / free lunch** в тестах.
- Миграции: упрощение baseline, параметры `-BaselineOnly` / `-SkipBaseline` в `migrate.ps1`.

### Инфраструктура docs / агентов

- Снимок метрик репозитория: [`PROJECT_META.md`](../../foundation/PROJECT_META.md).
- Pre-alpha triage: часть фидбека переведена в статус «ждём PA-T2».

## Что проверить вручную (smoke)

**Капитал**

- [ ] Доходы/Расходы раскрываются; с главной «Весь капитал →» открывает вкладку.
- [ ] Переключение **Детали ↔ Действия** без потери данных.
- [ ] Из «Детали» кнопка «+ Добавить» в пустой секции → «Действия» + нужный sheet.
- [ ] Депозит / облигации / страховка / шаблон актива / ипотека / кредит — toast, обновление баланса.
- [ ] Закрытие позиции и отмена полиса — confirm, затем результат.

**Регрессия рядом**

- [ ] Закрытие месяца, зарплата, подушка на главной.
- [ ] Цели победы на дашборде после выполнения шагов tutorial.
