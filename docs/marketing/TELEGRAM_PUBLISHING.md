# Публикация постов в Telegram

Money Quest Mini App и **бот для канала** — разные сущности. Посты в канал/группу делаются через **Telegram Bot API**, не через TMA SDK.

## Можно ли публиковать через API?

**Да**, в **канал** или **супергруппу**, где бот — администратор с правом публиковать сообщения.

**Нельзя** штатным Bot API:

- постить от вашего **личного** аккаунта как от пользователя;
- управлять Mini App от имени этого же бота без отдельной настройки (бот ≠ TMA).

Для личного профиля смотрят MTProto/userbot — это отдельный риск, ToS и поддержка; **не рекомендуем** для маркетинга проекта.

## Что нужно один раз

1. **Бот:** [@BotFather](https://t.me/BotFather) → `/newbot` → сохранить **token** (только в секретах, не в git).
2. **Канал:** публичный `@channel` или приватный с id `-100…`.
3. **Права:** канал → Administrators → добавить бота → включить **Post messages** (и при необходимости Edit).
4. **Chat id:** для публичного канала часто `@channelusername`; для приватного — `getUpdates` после первого сообщения в канале или сервисы вроде `@getidsbot`.

## Как отправить пост (вручную или скриптом)

**Только текст:**

```http
POST https://api.telegram.org/bot<TOKEN>/sendMessage
chat_id=@YourChannel
text=...
parse_mode=HTML
```

**Текст + картинка** (как в наших артефактах):

```http
POST https://api.telegram.org/bot<TOKEN>/sendPhoto
chat_id=@YourChannel
photo=@image.png
caption=...   # тело поста, до 1024 символов
parse_mode=HTML
```

Ограничения: caption до **1024** символов; фото — по лимитам Bot API (сжатие, размер файла).

## Автоматизация из репозитория (опционально, позже)

| Вариант | Плюсы | Минусы |
|---------|--------|--------|
| Ручная выкладка из `docs/marketing/posts/` | Без секретов в CI | Нет расписания |
| Локальный скрипт `curl` / Python + `.env` | Быстрый клик после «Утверждено» | Token на машине |
| GitHub Action с secret `TELEGRAM_BOT_TOKEN` | После merge в main | Нужен CI, review секретов |

Скилл **social-changelog-posts** по умолчанию **не** шлёт в API — только готовит `post.md` + `image.png`. Автоотправку включать отдельным решением.

## Планировщик

В Bot API **нет** встроенного «опубликовать в 18:00». Варианты: cron на сервере, GitHub Actions `schedule`, сторонние сервисы (Postpone, и т.д.).

## Связь с Mini App

- Тот же бренд-бот может открывать TMA через `Menu Button` / `Web App` — это **не** публикация постов.
- Deep link на игру в CTA поста — обычная ссылка `https://t.me/YourBot/app` (формат уточнять по вашей настройке бота).

## Безопасность

- Token только в `.env` / CI secrets; добавить `.env` в `.gitignore`.
- Не коммитить `TELEGRAM_BOT_TOKEN` и не вставлять в чат с агентом.
- Минимальные права бота в канале.
