# Трекер тем для соцсетей

Одна строка = одна **логическая тема** (фича или связный пакет коммитов).

| Тема / фича | Коммиты / контекст | Артефакт | Утверждено | Опубликовано |
|-------------|-------------------|----------|:-----------:|:------------:|
| *(пример)* Каталог недвижимости: owned / income / leased | `0029`, idea `real-estate-asset-catalog` | — | | |
| *(пример)* Стартовые шаблоны без жилья в blueprint | `0030`, ladder баланса | — | | |

## Правила колонок

| Колонка | Кто | Значение |
|---------|-----|----------|
| **Утверждено** | Агент после вашего «утверждаем» | Текст и картинка в `posts/<slug>/`, не в `draft/` |
| **Опубликовано** | **Только вы** | Пост реально вышел в соцсети/канале |

Отметки: пусто · `да` · дата (`2026-05-23`) · ссылка на пост.

## Как добавлять строки

1. Скилл **social-changelog-posts** предлагает темы по `git log`.
2. При первом черновике — новая строка, **Артефакт** = `posts/draft/draft_YYYY-MM-DD_<slug>/`.
3. После согласования — **Артефакт** = `posts/YYYY-MM-DD_<slug>/`, **Утверждено** = да.
4. После вашей публикации — **Опубликовано** = да (или дата).
