# ТВОЙ ХОД — user flows и болевые точки (TMA)

Документ для приоритизации UX: кратко **потоки**, **типичные ошибки пользователя** и **что уже сделано / что брать следующим**.

---

## Ключевые сценарии

| № | Цель пользователя | Краткий поток |
|---|-------------------|---------------|
| 1 | **Войти и попасть в игру** | Логин → меню старта → новая игра / загрузка → экран игры |
| 1a | **(Цель)** Выбрать тип новой партии | Экран вида **`NewProfileKindScreen`**: **Game** или **Plan** (в G1 Plan неактивен) → для **Game** — **`GameTemplatePickScreen`** (каталог шаблонов из API); для **Plan** (цель) — мастер ввода или префилл из **стартового снимка** |
| 2 | **Ориентироваться в периоде** | Видеть номер периода, статус «месяц открыт», закрыть месяц кнопкой |
| 3 | **Получить зарплату** | Главная → «Забрать зарплату» до конца периода |
| 4 | **Решить финансовые ситуации** | Блок событий периода → выбор (кнопки / свайп) → результат на балансе |
| 5 | **Управлять подушкой** | Ввод суммы → пополнение / снятие → обратная связь |
| 6 | **Смотреть финансовую картину** | Финансы → инвестиции / страховки / активы и долги |
| 7 | **Завершить период без сюрпризов** | **«Закрыть месяц»** → предупреждение о зарплате при необходимости → экономика месяца |
| 8 | **Выйти / новая партия** | Меню → выход или новый игрок; в списке сохранений — бейджи **`game` / `plan`** и **сложность** для игровых (см. [evolution §II](../vision/ideas/tvoy-hod-evolution-after-mvp.md)) |

---

## Типичный заход и цель по времени

- **Факт сейчас:** большинство заходов короткие, порядка **1–3 минуты** — один-два решения между отвлечениями TMA.
- **Куда сознательно растём** (контент, прогрессия, события): **10–15 минут** «плотного» захода как ориентир ёмкости, без принуждающего «сидеть в приложении» сверх уже заданной длины периода.
- Продуктовые рамки (ЦА, жёсткость последствий, избегаемые темы, копирайт событий): **[`TARGET_PLAYER_AND_SESSION.md`](TARGET_PLAYER_AND_SESSION.md)**.

## Болевые точки → ответ продукта

| Боль | Почему больно | Действие в продукте |
|------|----------------|---------------------|
| «Не понимаю границ приложения» | Контент сливался с чатом, нет вертикальной рамки | Рамка колонки `#root`, вторичный фон `body`, шапка с лого и названием |
| «Низ закрывает пол-экрана» | Три огромные кнопки таббара | Компактный таббар: `size="s"`, уменьшенные паддинги, min-height ~42px, sticky с blur |
| «События занимают экран» | Блок на главной мешал обзору | Кнопка **«События (N)»** в шапке → полноэкранный слой только по запросу, слайдер карточек (без общего скролла списка) |
| «Страшный/крупный шрифт» | Наследование от лендинговых переменных | Базовый текст **15px** в `#root`, системный шрифт стека Telegram-платформы |
| «Не видно акцента ТВОЙ ХОД» | Только тема TG | Переменные `--mq-*` как fallback; фиолетовый бейдж на блоке событий; логотип ТВОЙ ХОД |
| «После блокировки экрана баланс/период сбились» | WebView TG; клиент не resync при возврате | **PW1:** resync при `visibilitychange` (без auto-next по времени, TB1) — [PLAN_pwa-standalone](../plans/PLAN_pwa-standalone.md) |

## Принципы разработки TMA здесь

- **Telegram UI** для форм, списков, модалей подтверждения, спиннеров — не подменять целиком.
- **Кастом** только там, где библиотека не задаёт паттерн: карточки событий, брендовые кнопки «да/нет», шапка.
- **Не ломать `tg-theme-*`**: фон текста брать из темы клиента; брендовый цвет — точечный акцент.

- [x] Выбор **Game / Plan** (Plan — заглушка) и поток **шаблонов** вместо `light` / `hardcore` (см. [`SPEC_game-plan.md`](../specs/features/SPEC_game-plan.md)).

- [ ] Локализованные подписи типов активов / событий (не сырое поле API).
- [ ] Прогресс цели победы визуально на главной (в т.ч. после перехода на **M из N** целей из шаблона).
- [ ] Пустые состояния вкладки «Финансы».
