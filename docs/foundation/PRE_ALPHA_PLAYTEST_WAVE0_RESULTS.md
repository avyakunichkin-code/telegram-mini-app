# Pre-Alpha: пилот wave-0 (1 тестер)

**Дата:** 2026-05-25  
**Источник:** устный разбор с автором после сессии  
**Прогресс:** дошёл до **3–4 периода** (цель пилота достигнута)

## Сводка

| Метка | Наблюдение |
|-------|------------|
| **UX** | Не сразу понятен цикл «период → зарплата → подушка → следующий период»; guided coach частично помог, но первые минуты — трение |
| **PERF** | Ощущение «медленно» при загрузке и после действий (ожидание ответа сервера) |
| **POSITIVE** | Дошёл до 3–4 периода без блокера «не могу продолжить» |

## Гипотезы (технические)

1. **Bootstrap** на каждый запрос синхронизировал полные каталоги событий и достижений в БД — лишние сотни SQL на read-path.
2. **Онбординг:** 10 с «практики» без явного обратного отсчёта; длинные тексты; нет одной фразы «план первого месяца» в начале.
3. **Холодный старт API** (Render) — вне кода; см. [`docs/ops/DEPLOY.md`](../ops/DEPLOY.md).

## Принятые правки (2026-05-25)

| # | Изменение |
|---|-----------|
| 1 | Early-return в `ensure_mvp11_event_catalog` / `ensure_achievement_catalog`, если каталог уже полный |
| 2 | Онбординг: короче копирайт, план «зарплата → подушка → период», практика **6 с** с обратным отсчётом |
| 3 | `onboarding_state: started` при первом показе coach |
| 4 | `refreshGameState` без полноэкранного loading (только первый вход) |
