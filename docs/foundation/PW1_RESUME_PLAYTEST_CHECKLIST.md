## 1. Сценарии (обязательные)

Отметить **PASS** / **FAIL** / **N/A** и одну строку заметок.

### A — Короткая блокировка (1–2 мин)

| Шаг | Действие | Ожидание | Результат |
|-----|----------|----------|-----------|
| A1 | Запомнить **таймер** и **cash** на дашборде | — | |
| A2 | Заблокировать экран на **1–2 мин** (или свернуть TG) | — | |
| A3 | Разблокировать, вернуться в ту же сессию TMA **без** перезапуска бота | В течение ~2 с таймер **согласован** с реальностью (не «замёрз» на старом значении) | |
| A4 | Cash / период на дашборде **не устарели** (нет расхождения с вкладкой «Финансы» после pull) | Данные обновились или остались корректными | |

## 2. Критерии приёмки фазы 0

Все **обязательные** сценарии A–D — **PASS** на минимум **двух** устройствах (iOS + Android или два Android).

| Критерий | Порог |
|----------|--------|
| Таймер после unlock | Расхождение с ощущением «реального» времени ≤ **~5 с** (визуально; точнее — сравнить с `GET /api/game/time` в Network) |
| Двойной конец периода | **0** случаев на прогон C |
| Потеря сессии | **0** принудительных re-login после unlock |
| BLOCKER по resume | **0** открытых после прогона |

## 3. Верификация API (для модератора, опционально)

При сценарии A3 открыть DevTools / прокси (если доступно) или повторить в **браузере** с тем же аккаунтом:

1. После unlock — запрос `GET /api/game/bootstrap`.
2. Поле `time.seconds_until_next_period` должно соответствовать UI таймеру (±2 с).
