# Арендуемое жильё и поражение «негде жить» (idea)

**Статус:** идея, не в MVP кода; связано с [real-estate-asset-catalog.md](real-estate-asset-catalog.md).

## Суть

Игрок может жить в **арендуемом** жилье (`estate_role = leased`): каждый период платит **аренда + ЖКУ**, без `asset_value`.

Если есть **собственное** жильё (`owned` с `kind` home/house/…) — можно **прекратить аренду** и убрать ежемесячные платежи по lease (механика UI/API — позже).

## Поражение

- **3 периода подряд** без жилья (нет ни активной аренды, ни собственного жилья для проживания) → **game over**.
- Каждый период без жилья: красная подсказка в UI:  
  **«Вам негде жить. Через N период(-а) вы проиграете.»**  
  где `N = 3 − streak`.

## Задел в данных

- Каталог `lease_*` с `monthly_rent_cost`, `monthly_utilities_cost`.
- На профиле: флаг/счётчик `homeless_period_streak` (или использовать существующий механизм поражения).
- Отдельно от `has_tenants` на **доходной** недвижимости (там риск потери **дохода**, не крыши над головой).

## Не путать

| Ситуация | Эффект |
|----------|--------|
| Жильцы съехали с доходной квартиры | 2–3 периода **0 дохода** от актива, `has_tenants=false` |
| Нет аренды и нет своего жилья | **Поражение** через 3 периода |

События и UI — после каталога `0029`.
