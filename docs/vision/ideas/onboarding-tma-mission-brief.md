# Онбординг TMA — Mission Brief + первый период

## Problem Statement

**Как помочь новичку в Money Quest за первый заход понять цикл «период → действия → конец месяца» и зачем нужны зарплата и подушка — без ощущения финтех-лекции и без перегруза перед Pre-Alpha плейтестом?**

Контекст: TMA, сессии **1–3 мин** сейчас, цель роста до **10–15 мин** плотной игры ([`TARGET_PLAYER_AND_SESSION.md`](../../foundation/TARGET_PLAYER_AND_SESSION.md)). В БД уже есть `game_profiles.onboarding_state`, UI онбординга нет.

---

## Recommended Direction

**Гибрид «Mission Brief» (3 экрана) + максимум 2 контекстные подсказки в периоде 1** — не отдельное приложение-обучалка.

### Почему не другие паттерны

| Паттерн | Плюс | Минус для нас |
|---------|------|----------------|
| Длинный wizard / 10–15 мин туториал | Полное объяснение | Убивает «игру», не влезает в TMA-сессию; анкета Closed Alpha — не сейчас |
| Только coach marks по UI | Быстро внедрить | Не объясняет «что такое период» до появления интерфейса |
| Только progressive unlock (level gates) | Уже есть в M11 | Не заменяет смысл подушки/зарплаты |
| Видео / Lottie-ролик | Эмоция | Тяжело поддерживать, слабая доступность, лишний вес билда |
| Diegetic NPC-диалоги | Сильная игра | Дорого по контенту для Pre-Alpha; оставить на Closed Alpha |

### Решения (зафиксировано 2026-05-19)

| Вопрос | Решение |
|--------|---------|
| Точка показа | **После загрузки overview** — оверлей на `GameScreen`, не до API |
| Волны | **Сначала только Mission Brief**; coach marks — волна 2 после утверждения |
| Повтор при 2-й игре | **Не показывать** автоматически |
| Повтор вручную | Кнопка **«Повторить обучение»** в меню |
| Тон | **Игровой**; наставник — персонаж **Монетка** ([`CHARACTER_MONETKA.md`](../../reference/CHARACTER_MONETKA.md)) |
| Утверждение | **design-lab** раунд 1 (варианты A–F) → явное «утверждаем X» → только потом код |

### Что делаем

1. **Оверлей на `GameScreen`** после первого `overview` — **Mission Brief** из **3 карточек** (свайп или «Далее»), MQX, **Монетка** в утверждённой компоновке.
2. **Coach marks** — **не в волне 1**; отдельный раунд design-lab / spec после приёмки брифа.
3. **Состояние** — `onboarding_state`: `draft` → `brief_done`; меню сбрасывает показ брифа без смены state (или `replay` — в spec).
4. **Визуал** — [`design-lab/onboarding-brief/`](../../../design-lab/onboarding-brief/) → MQX → prod.

### Три шага брифа (копирайт черновик)

| # | Заголовок | Суть (1–2 предложения) |
|---|-----------|-------------------------|
| 1 | **Месяц за месяцем** | Игра идёт **периодами**. Таймер можно **паузить**. Кнопка **«Дальше»** закрывает месяц: списания, доходы, итог. |
| 2 | **Зарплата и подушка** | **Зарплату** забираешь **сам** до конца периода — иначе за этот месяц не придёт. **Подушка** — запас на чёрный день; пополнять выгодно, когда есть лишние деньги. |
| 3 | **Ситуации и цель** | В шапке — **«События»**: несколько выборов за период. На главной видно **цель** (подушка, долги, поток). Ошибаться можно — важно понять, *почему* цифры изменились. |

Кнопки: **«Пропустить»** (всегда) · **«Начать первый месяц»** на последнем шаге.

## Key Assumptions to Validate

- [ ] **80%** плейтестеров после брифа доходят до **3–4 периода** без вопроса «что нажимать» (опрос Pre-Alpha).
- [ ] Пропуск брифа **не хуже** по completion, чем просмотр — иначе бриф слишком длинный.
- [ ] Двух подсказок в периоде 1 **достаточно**; третья (вклад «Финансы») не нужна до периода 2–3.

---

## MVP Scope (эпик O1, волна Pre-Alpha)

**In:**

- 3-карточный Mission Brief (RU), skip, сохранение `onboarding_state`.
- Показ только для **нового** game-профиля (`brief_done` / `complete` не показывать).
- `GET /api/finance/overview` или профиль отдаёт `onboarding_state`; `PATCH` прогресса (минимальный контракт).
- Design-lab → MQX → smoke в TMA.

**Out (явно):**

- Обучение инвестиций, страховок, аналитики, достижений.
- Блокирующий туториал «нельзя нажать, пока не прочитал».
- Отдельный онбординг для **Plan** (мастер Plan — другой эпик).
- Персонаж-наставник с ветвлением диалога.
- Локализация EN.

---

## Not Doing (and Why)

- **Единый 10-минутный сценарий** — цель Closed Alpha, не Pre-Alpha.
- **Объяснение tier/XP на брифе** — мета появится в игре; HUD уровня уже на дашборде.
- **Coach marks на все 4 таба** — когнитивный шум; финансы открываются по tier и любопытству.
- **Повтор брифа при каждой новой игре без флага** — только первый профиль или явный «Повторить обучение» в меню (P2).
