# Money Quest — user flows и болевые точки (TMA)

Документ для приоритизации UX: кратко **потоки**, **типичные ошибки пользователя** и **что уже сделано / что брать следующим**.

---

## Ключевые сценарии

| № | Цель пользователя | Краткий поток |
|---|-------------------|---------------|
| 1 | **Войти и попасть в игру** | Логин → меню старта → новая игра / загрузка → экран игры |
| 2 | **Ориентироваться во времени** | Видеть период, таймер, паузу/продолжение, переход месяца |
| 3 | **Получить зарплату** | Главная → «Забрать зарплату» до конца периода |
| 4 | **Решить финансовые ситуации** | Блок событий периода → выбор (кнопки / свайп) → результат на балансе |
| 5 | **Управлять подушкой** | Ввод суммы → пополнение / снятие → обратная связь |
| 6 | **Смотреть финансовую картину** | Финансы → инвестиции / страховки / активы и долги |
| 7 | **Завершить период без сюрпризов** | «Дальше» → предупреждение о зарплате при необходимости → экономика месяца |
| 8 | **Выйти / новая партия** | Меню → выход или новый игрок |

## Болевые точки → ответ продукта

| Боль | Почему больно | Действие в продукте |
|------|----------------|---------------------|
| «Не понимаю границ приложения» | Контент сливался с чатом, нет вертикальной рамки | Рамка колонки `#root`, вторичный фон `body`, шапка с лого и названием |
| «Низ закрывает пол-экрана» | Три огромные кнопки таббара | Компактный таббар: `size="s"`, уменьшенные паддинги, min-height ~42px, sticky с blur |
| «События занимают экран» | Блок на главной мешал обзору | Кнопка **«События (N)»** в шапке → полноэкранный слой только по запросу, слайдер карточек (без общего скролла списка) |
| «Страшный/крупный шрифт» | Наследование от лендинговых переменных | Базовый текст **15px** в `#root`, системный шрифт стека Telegram-платформы |
| «Не видно акцента Money Quest» | Только тема TG | Переменные `--mq-*` как fallback; фиолетовый бейдж на блоке событий; лого MQ |

---

## Принципы разработки TMA здесь

- **Telegram UI** для форм, списков, модалей подтверждения, спиннеров — не подменять целиком.
- **Кастом** только там, где библиотека не задаёт паттерн: карточки событий, брендовые кнопки «да/нет», шапка.
- **Не ломать `tg-theme-*`**: фон текста брать из темы клиента; брендовый цвет — точечный акцент.
