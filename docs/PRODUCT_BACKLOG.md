## Backend и API

- [ ] P1 **Alembic / версионируемые миграции** — вместо только `create_all` + разовые SQL; снижает риск дрейфа схемы между окружениями.
- [ ] P2 **Единый формат ошибок API** — стабильные `code`, человекочитаемый `message`, при необходимости `details` для фронта.
- [ ] P1 **Идempotency ключи** для денежных действий (`claim-salary`, перевод в подушку, покупки) — защита от двойных тапов в TMA.
- [ ] P2 **Лимиты частоты (rate limit)** по `user_id` / IP для чувствительных эндпоинтов.
- [ ] P2 **Журнал аудита** — кто/когда сменил период, крупные операции (опционально, privacy-aware).
- [ ] P1 **Валидация старта игры** — согласованные правила сумм, количества активов/долгов, сообщения без «немого» 500.
- [ ] P2 **Версионирование API** (`/api/v1/…`) когда появится внешний клиент или breaking changes.
- [ ] P3 **OpenAPI/SDK** — автогенерация клиента для фронта или партнёрских интеграций.

## Экономика и механики игры

### Уже заложено в easy (ориентир по коду — при расхождении поправить чекбокс)

- [x] **До трёх pending-событий на период** в режиме `light` (`ensure_period_events`, ответ `GET /pending` с полем `events`).
- [x] **Core loop периода** — таймер, конец периода, переход (`process_period_end` и связка с `/time/next`).
- [x] **Зарплата по кнопке**, пропуск периода — последствия (easy).
- [x] **Обязательства** — списание, нехватка → `overdue_amount` / счётчики.
- [x] **Активы** — обслуживание, `monthly_income` в конце периода.
- [x] **Easy: события** — pending / choose.
- [x] **Easy: инвестиции** — депозит (капитализация), облигации (купон).
- [x] **Easy: страховки** — премии по периодам, связка с событиями покрытия (по текущей реализации MVP).
- [x] **Типовые активы / шаблоны** и создание из шаблона.
- [x] **Победа MVP** — подушка, просрочки, чистый денежный поток (`overview`).
- [x] **Поражение** — связка с `negative_periods_count` (3 подряд в логике периода).

## UI / UX (TMA и онбординг)

### Реализовано / частично (отметьте неточности по факту)

- [x] **Русскоязычный HUD времени** — период, таймер, Играть/Пауза/Дальше.
- [x] **Подтверждение перехода периода** — модал вместо `confirm`, предупреждение о незабранной зарплате.
- [x] **Ошибка загрузки игры** — текст + «Повторить загрузку».
- [x] **Контейнер `#root` под TMA** — ограничение ширины, `tg-theme`, safe-area учтены в ключевых местах.
- [x] **Финансы: сегменты** (инвестиции / страховки / шаблоны / списки).
- [x] **Список обязательств: просрочка** суммарно по долгу там, где есть данные из API.
- [x] **Список активов: доход** и тип (`kind`).
- [x] **Тосты вместо alert** на шагах сложности и при ошибках старта игры из базовых параметров.
- [x] **Лого MQ + переменные палитры бренда** в `#root` (fallback к `tg-theme`), рамка колонки приложения.
- [x] **Компактный таббар** и базовый **15px / системный стек** в колонке TMA.
- [x] **События easy: до 3 за период** с бэка + **кнопка в шапке** и **оверлей-слайдер карточек** (закрытие ×, зелёный/красный при двух исходах).
